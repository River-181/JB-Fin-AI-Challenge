## 1. 누구의 문제인가 — 대상 사용자 정의

문제의 주체를 "고객"과 "담당자" 두 층으로 분리해서 본다. 이 분리는 제품 설계의 출발점이다 (챗봇이 아니라 운영 콘솔을 만드는 이유).

### 1.1 1차 사용자 — 지역 금융 현장 담당자
- **RM(기업·소상공인 담당)**: 1인이 수십~수백 차주를 보유. 위험 징후를 케이스별로 추적할 시간 부족.
- **기업금융 심사역**: 신청서·재무자료·거래정보를 종합해 판단해야 하나 자료가 분산.
- **사후관리 담당**: 실행 후 차주 상태 변화를 지속 추적해야 하는데 누락이 발생.
- **준법·리스크 담당**: 고객 대상 행동의 과장·조건 확정 표현·개인정보 노출을 사전 차단해야 함.

이들은 "AI가 대신 답해주는 것"보다 **근거 있는 판단 초안 + 승인·감사 가능한 행동안**을 더 필요로 한다. 금융 현장의 의사결정은 책임소재가 사람에게 귀속되기 때문이다.

### 1.2 최종 수혜자 — 지역 금융 고객
- 전북·광주/전남 기반 **소상공인·개인사업자** (히어로 케이스: 전주 중앙로 카페 `JBG-104`, riskScore 88, 운전자금 1.8억·카드매출 둔화).
- **전세 계약 고객** (특히 청년·고령) — 전세가율·권리관계·보증보험 정보가 흩어져 사전 판단이 어려움.
- **금융사기 노출 고객** (특히 50대 이상) — 비정상 콜백·외부 접촉을 조기에 차단해야 하는 대상.

## 2. 어떤 상황에서 문제가 발생하는가 — 사용 상황

문제는 "정보 부족"이 아니라 **"신호 분산 + 조기 종합 판단 시간 부족"**이다. 담당자가 마주하는 전형적 상황:

1. 위험 신호가 **기사·정책공고·시세·등기·상담기록·거래·사기경보** 등 서로 다른 출처에 흩어져 있다.
2. 차주 단위로 이 신호들을 **연체 발생 이전에** 모아 판단할 운영 도구가 없다.
3. 판단했더라도 다음 행동(대환 검토, 정책금융 매칭, 고객 안내, 준법 검토)이 **수작업으로 단절**된다.
4. 실행 이후 사후관리 추적이 사람의 기억·엑셀에 의존해 **누락**된다.

→ 핵심 상황은 *"분산된 위험 신호를 연체 전 단계에서 종합하여, 승인 가능한 다음 행동으로 끊김 없이 연결하지 못하는"* 운영 공백이다.

> 대응: 1.2 (대상·사용 상황), 1.3 (병목·리스크)

## 3. 세 개의 Pain 도메인 (검증된 통계)

> 아래 수치는 모두 canon §10 검증 집합. 상세 출처: [`docs/01_research/pain-point-evidence.md`](../05_리서치/pain-point-evidence.md). canon에서 ⚠️ 미검증으로 표기된 단일지표는 본 문서에서 인용하지 않는다.

### 3.1 소상공인 자금압박 (히어로 도메인)
- 자영업자 대출 잔액 **1,064.2조원**, 취약 자영업자 **42.7만명(13.7%)** (한국은행, 2024말).
- 자영업자 연체율: 전체 **1.67%** / 비은행 **3.43%** / 취약 차주 **11.16%**.
- 2024년 폐업 **100.8만명** (첫 100만 돌파), 폐업률 **9.04%**, 사업부진 사유 **50.2%** (국세청).
- **함의**: 금리 부담과 매출 둔화가 동시에 오는데, 연체 전 단계 개입이 늦으면 폐업·부실로 직결된다. 담당자에게는 사후관리 누락·대응 지연이 가장 큰 리스크다.

### 3.2 전세사기·보증 리스크
- 피해자 누적 인정 **약 3.9만건** (국토부, 2026), HUG 사고액 **2024년 4.49조원** (역대 최고).
- 단, 2025년 보증 강화로 사고액은 급감(추세 반전) — 이는 *사전 통제가 작동하면 줄어든다*는 반증이자, 제품의 사전 체크리스트 가치를 뒷받침한다.
- **함의**: 전세가율·권리관계·보증보험 정보가 분산되어 고객·담당자 모두 계약 전 판단이 어렵다.

### 3.3 금융사기 (보이스피싱)
- 2025년 1~11월 피해액 **1.13조원(+56% YoY)**, 1건 평균 **5,301만원**, 50대 이상 **53%** (금융위·경찰청).
- **함의**: 비정상 콜백·외부 접촉을 조기에 감지하고, 고위험 시 외부 행동을 자동 차단하는 통제가 필요하다.

> 대응: 1.3 (Pain Point 설득)

## 4. 왜 JB금융그룹·금융업 관점에서 중요한가

> 상세 출처: [`docs/01_research/jb-group-business-facts.md`](../05_리서치/jb-group-business-facts.md)

1. **지역 밀착 익스포저의 집중**: JB는 전북·광주/전남에서 소상공인·개인사업자·전세 고객 접점이 강하다. 전북은행은 소상공인 **대환대출 취급 2024년 687억으로 1위**, 광주은행은 광주·전남 상생금융을 확대한다. 차주 리스크 조기 탐지는 곧 **건전성 + 상생금융 + 고객 유지**로 동시에 직결된다.
2. **그룹 공표 추진 방향과의 정합**: JB금융그룹은 2025-12-26 **네이버클라우드와 MOU**를 맺고 기업여신 **상담→심사→사후관리** 전반의 AI 적용을 단계적으로 검토한다고 공표했다 (MOU·공표한 추진 방향). 본 제품은 그 방향의 **실행 수단**이다.
3. **그룹 최우선 전략과의 정합**: 김기홍 회장은 2026년을 **"AX(AI 전환) 원년"**으로 선언했다.
4. **규모는 작지만 효율로 승부하는 명분**: FY2025 그룹 순익 **7,104억(사상 최대)**, ROE **12.4%**(7년 연속 두 자릿수), 지방 3대 지주. 대형 지주 대비 IT 자원 제약이 있으므로 **경량 AI Agent 콘솔로 운영 효율을 극대화**하는 전략이 타당하다. (그룹 총자산 수치는 canon에서 미검증이므로 인용하지 않는다.)

> 대응: 1.4 (중요도·JB 관점), 2.1 (사업 연결성 — 사전 연결)

## 5. 문제 정의 → 서비스 과제로의 전환

문제는 추상적 "AI 도입"이 아니라 **명확한 운영 과제**로 환원된다:

> *"분산된 위험 신호를 연체 전 단계에서 케이스로 종합하고, 전문 AI Agent가 판단·행동 초안·검증을 수행하되, 고객 대상 행동은 사람 승인 전까지 차단하는 승인·감사형 운영 콘솔."*

이는 신규 시스템을 통째로 도입하는 것이 아니라, **기존 RM 업무 흐름(상담→심사→사후관리)에 삽입되는 운영 레이어**다. 그래서 실현 가능성이 높다:

| 문제 요소 | 서비스 과제로의 전환 |
| --- | --- |
| 신호 분산 | `Case` 단위 종합 + 출처 칩(공공데이터/시뮬레이션/추정) |
| 조기 판단 시간 부족 | 전문 Agent의 판단·근거·confidence 자동 생성 (Triage time 50% 단축 목표) |
| 행동 단절 | `AgentRun → Approval` 으로 다음 행동을 끊김 없이 연결 |
| 사후관리 누락 | 케이스 상태·SLA 큐잉 (사후관리 누락 0건 목표) |
| 책임소재·설명가능성 | 모든 판단을 Evidence·Audit으로 연결 (근거 연결 100%) |

운영 계약 `Case → AgentRun → Agent → Skill → Evidence → Approval → Audit` (canon §8) 이 그 전환의 구체적 골격이다.

> 대응: 1.5 (서비스 과제 전환 가능성)
