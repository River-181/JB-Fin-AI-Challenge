## 1. 왜 자유주제에 부합하는가

대회의 자유주제는 **"JB금융그룹 연계 AI Agent 서비스"** (JB 미래사업 AI)다. LocalGuard OS는 다음 세 가지로 그 취지에 정확히 부합한다:

1. **JB 연계성**: 전북은행·광주은행·JB우리캐피탈의 지역 금융 접점(소상공인·개인사업자·전세·금융사기 고객)을 직접 대상으로 한다. JB가 공표한 추진 방향(네이버클라우드 MOU의 상담→심사→사후관리 AI)의 실행 콘솔이다. (MOU·공표한 추진 방향)
2. **AI Agent 서비스**: 단일 모델 호출이 아니라, 판단·행동 초안·검증을 갖춘 **멀티 에이전트 운영체계**다 (운영 조율·상환위험 분류·정책금융 매칭 등 14종 운영 에이전트).
3. **미래사업 정합**: 김기홍 회장 **"AX 원년"** 선언, 3사 공동 AI 거버넌스 방향과 정합하는 **그룹 AX의 구체적 실행 수단**이다.

→ 즉 자유주제의 "자유"를 임의 도메인으로 해석하지 않고, **JB의 실제 사업 현실(지역 금융 리스크 + 공표한 AI 방향)에 최적화된 Agent OS**로 좁혔다. 이것이 주제 부합의 핵심 논거다.

> 대응: 1.1 (자유주제 취지 부합)

## 2. 이전 Agent-OS 패턴에서 가져온 것 / 버린 것

본 제품은 기존에 검증된 **Agent 운영체계(Agent-OS) 패턴**을 토대로 한다. 그러나 가져온 것은 **시스템 구조뿐**이고, 도메인(학원/아카데미 운영)은 전부 버렸다. 이 구분을 명확히 하는 것이 자유주제 정당성의 핵심이다.

### 2.1 가져온 것 — 시스템 구조 (도메인 무관)
- 운영 계약 **`Case → AgentRun → Agent → Skill → Evidence → Approval → Audit`** (canon §8).
- **스킬 장착형 멀티 에이전트** 구조 (기능 추가 = 스킬 추가).
- **승인 게이트(L0–L4)** 와 **감사 원장(Audit Ledger)** 을 일급 객체로 두는 운영 모델.
- 4-zone 콘솔 UI, 케이스 보드/칸반의 상태 전환 hook.

### 2.2 버린 것 — 이전 도메인 (학원/아카데미)
- 학원 운영자 도메인의 데이터 모델·업무 흐름·용어는 **전부 제거**.
- 이전 도메인의 고객/업무 시나리오를 끌고 오지 않는다.

### 2.3 새로 정의한 것 — 금융 도메인
- 지역 금융 리스크 도메인의 케이스(소상공인 상환위험·전세 위험·금융사기).
- 금융 전용 에이전트 로스터 14종 + 사람 승인자 2종 + 승인 게이트 (canon §2).
- 금융 규제 정합 거버넌스(PII 비반출 4중 방어, 신용정보법 §40조의2 등).

| 구분 | 내용 | 자유주제 함의 |
| --- | --- | --- |
| 가져옴 | 운영체계 구조 (Case·AgentRun·Approval·Audit·Skill) | 검증된 구조 재사용 → 구현 실현성 ↑ |
| 버림 | 학원/아카데미 도메인 전부 | 자유주제를 JB 금융으로 재정의 |
| 신규 | 금융 리스크 도메인·에이전트·거버넌스 | JB 연계성·규제 정합 확보 |

> 메모: 발표·제출 시 "기존 시스템 구조를 금융 도메인에 이식했다"로만 설명하고, 이전 도메인(학원)을 노출하지 않는다 — JB 자유주제로서의 정체성 일관성 유지.

## 3. 스코프 정의 (In / Out)

명확한 경계는 4.1 일관성과 5.5 리스크 대응의 근거가 된다.

### 3.1 스코프 In — 무엇을 한다
- 분산된 위험 신호를 케이스로 종합.
- 위험 원인 분류·정책금융 후보 탐색·전세 위험 체크리스트·RM 안내문 초안·준법 검토.
- 승인 요청 라우팅(L0–L4)·감사 로그 기록.
- 고위험 금융사기 시 외부 접촉 차단·에스컬레이션 메모.
- 외부 LLM 활용 시 PII 토큰화·반출 스캔.

### 3.2 스코프 Out — 무엇을 하지 않는다 (자동화하지 않는다)
- **신용 판단·대출 승인**의 최종 결정 (사람 승인자가 수행).
- **법률 결론**의 확정 (준법 최종 승인자가 수행).
- **고객 대상 발송**의 무인 자동 실행 (승인 전 차단).
- **개인정보 포함 원본의 외부 전송** (4중 방어로 비반출).

### 3.3 MVP 스코프 (현재) vs 본선 스코프 (확장)
- **현재 MVP**: Vanilla JS/CSS/HTML 정적 앱. `python3 -m http.server` 구동, 네트워크 API 없이 브라우저 상태로 운영 루프 재현. 골든 패스 3종(`?demo=sme|jeonse|phishing`)이 deterministic하게 동작. (canon §9)
- **본선 목표**: 정적 함수 계약(`computeRiskDecision`·`buildDashboardData`·`auditChainRecords`·`moveCaseToColumn`)을 서버 API로 1:1 승격, RAG(공공/내부 문서)·Rule Engine(승인 레벨)·멀티 에이전트 오케스트레이션 구현.
