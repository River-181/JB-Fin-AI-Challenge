# 02_전략

JB LocalGuard OS의 대회 대응 전략, 자유주제 포지셔닝, 평가항목 대응 방향을 모아두는 공간입니다.

## 이 폴더의 역할

이 폴더는 "왜 이 주제가 JB금융그룹 자유주제에 맞는가"를 설명합니다. 제품 기능보다 먼저 문제 정의, JB 사업 연결성, 기존 HagentOS 계열 운영체계에서 무엇을 가져왔고 무엇을 버렸는지 정리합니다.

## 자유주제 포지셔닝

- 자유주제: JB금융그룹 미래 금융 서비스에 맞춘 AI Agent 운영체계
- 도메인: 지역 금융 고객의 위험 신호 탐지, 사후관리, 승인형 자동화
- 이전 대상: 학원 운영자 도메인이 아니라 `Case -> AgentRun -> Approval -> Audit` 시스템 구조
- 차별점: 전세사기 대응 `Jeonse Shield`와 지역 소상공인 pain radar를 하나의 Agent OS로 통합
- 안전성: 고객-facing 행동과 민감 금융 판단은 사람 승인 전 자동 실행하지 않음

## 전략 메시지

| 질문 | 답변 |
| --- | --- |
| 왜 챗봇이 아닌 Agent OS인가 | 금융권은 고객에게 바로 답하는 자동화보다 담당자 검토, 승인, 감사가 가능한 내부 운영체계가 더 도입 가능성이 높습니다. |
| 왜 JB인가 | 전북은행, 광주은행, JB우리캐피탈은 지역 금융 접점이 강하고, 소상공인/개인사업자/전세 고객의 위험 조기 감지와 사후관리에 직접 연결됩니다. |
| 무엇을 자동화하나 | 근거 수집, 위험 원인 분류, 정책금융 후보 탐색, 전세 위험 체크리스트, RM 안내문 초안, 승인 요청, 감사 로그를 자동화합니다. |
| 무엇을 자동화하지 않나 | 신용 판단, 대출 승인, 법률 결론, 고객 발송, 개인정보 포함 외부 전송은 자동 처리하지 않습니다. |

## 평가 대응 포인트

- 문제 정의: 지역 금융 고객의 위험 신호가 여러 출처에 흩어져 담당자가 조기 대응하기 어렵다는 pain point
- 사업 적합성: JB 계열사의 지역 금융, 기업대출 상담/심사/사후관리, 전세대출 상담 접점과 연결
- AI Agent 완성도: 여러 전문 Agent, Skill Registry, AgentRun, Approval Gate, Audit Ledger 구조
- 실현 가능성: 정적 MVP에서 케이스 선택, 실행, 승인, 로그 갱신이 재현됨
- 리스크 관리: 준법 검토, 개인정보 마스킹, 승인 정책, 외부 접촉 차단 원칙 내장
