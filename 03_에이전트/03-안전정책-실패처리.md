# 03 · 안전정책 · 실패처리 (Safety Policy & Failure Handling)

> 단일 출처: [`docs/_canon.md`](../_canon.md) §8 운영 계약. [Skill Registry](skill-registry.md) 안전 정책.
> 심사 대응: 5.5(환각·설명가능성·책임소재 리스크 대응 방향).

JB LocalGuard OS는 환각·불확실성·근거 부족을 "실패"가 아니라 **명시적으로 다루어야 하는 정상 상태**로 취급한다. 에이전트는 모르는 것을 추정하지 않고, 고객 대상 행동은 사람 승인 전까지 차단된다.

---

## 1. 불확실할 때 에이전트가 내야 하는 출력

에이전트는 확신이 없을 때 결론을 만들어내지 않는다. 대신 아래를 강제 출력한다.

| 상황 | 금지 행동 | 필수 출력 |
| --- | --- | --- |
| 근거 부족 | 결론 단정 | `judgment="미확정"` + 부족한 Evidence 명시 + Evidence 요청 |
| 출처 신뢰도 낮음 | 낮은 출처로 행동 초안 | `confidence` 하향 + 출처 한계 기록 + 사람 검토 이관 |
| 모르는 정보 | 추정·창작(환각) | "추정하지 않음" 선언 + `nextChecks`에 확인 항목 |
| 법적/금융 판단 경계 | "안전하다/확정" 표현 | "검토 필요 신호"로만 표현 + L3 이관 |

> 원칙(캐논 §8 / 프롬프트 설계 원칙): 에이전트는 결론만이 아니라 **근거·불확실성·다음 확인 항목**을 남긴다. 전세/법률/금융 판단은 "확정"이 아니라 "검토 필요 신호"로 표현한다. 모르는 정보는 추정하지 않고 Evidence 요청 또는 사람 검토로 넘긴다.

---

## 2. 환각 방지 장치

| 장치 | 작동 방식 | 담당 |
| --- | --- | --- |
| Evidence 강제 연결 | 모든 판단은 내부 이벤트·출처 링크 필수(traceability 100%) | 전 에이전트 |
| 출처 랭킹 | 출처 신뢰도·최신성 평가 후 사용 | 위험신호 조기감지 에이전트 |
| 표현 제한 | 보장성·확정 문구를 검토 안전 문구로 교체 | 준법 검토 에이전트(`claim-limiter`) |
| 공식 문서 확인 의무 | 권리관계·세금·보증 조건은 공식 원문으로 확인, 단정 금지 | 등기 권리 분석 에이전트, 계약 체크리스트 에이전트 |
| 초안 표기 | 고객-facing 문장은 기본적으로 초안, 승인 전 미발송 | RM 보좌 에이전트, 은행 연계 에이전트 |

## 3. 에스컬레이션 규칙 (L0–L4)

승인 레벨은 캐논 §8 고정. 산출물은 요구 레벨이 충족될 때까지 외부 실행되지 않는다.

| 레벨 | 의미(캐논 §8) | 실패/불확실 시 동작 |
| --- | --- | --- |
| **L0** | 관찰/로그 | 신호 기록 후 종료. 행동 없음 |
| **L1** | RM 검토 | 초안을 RM 검토 큐로 |
| **L2** | RM 승인 | 1-click 승인 전까지 보류 |
| **L3** | 준법/법률 | 준법·법률 사람 판단으로 이관 |
| **L4** | **차단·상위검토** | 외부 행동 전면 차단 + 상위 검토 |

- 고위험 fraud는 **이상거래 탐지·차단 에이전트**가 모든 외부 행동을 차단(L4)한다.
- 한 Case에서 더 높은 레벨이 발동되면 보수적으로 더 높은 레벨이 우선한다.

---

## 4. Do-not-contact (외부 접촉 차단)

- fraud 위험 또는 준법 위험이 높으면 `do-not-contact-rule`이 고객 대상 출력을 즉시 중단한다.
- 차단된 행동은 에스컬레이션 메모와 함께 감사 원장에 기록되고, 상위 검토 전까지 재개되지 않는다.
- 고객 대상 경고 메시지조차 준법 검토 에이전트 통과가 전제다.

---

## 5. 준법 검토 루프 (Compliance Review Loop)

```
RM 보좌 에이전트 (초안)
        │
        ▼
준법 검토 에이전트  ──(수정 요청)──▶  RM 보좌 에이전트 (재초안)
        │                                      │
   (통과)                                   재검토
        ▼
   승인 게이트 (L1/L2/L3) ──▶ 승인 후에만 외부 실행
```

- 준법 검토 에이전트는 금지 표현·개인정보 노출·오인 유발 혜택 문구를 잡는다(L3, mandatory).
- 통과하지 못한 초안은 외부로 나가지 못한다. 재초안→재검토 루프를 반복한다.

## 6. 책임소재 · 설명가능성 (5.5)

| 질문 | 답 |
| --- | --- |
| 누가 판단했나 | Audit 원장에 에이전트·시각·승인자 기록 |
| 무엇에 근거했나 | 모든 판단에 Evidence 링크(traceability 100%) |
| 누가 책임지나 | 고객 대상 행동은 사람 승인자(RM 최종 승인자 / 준법 최종 승인자)가 최종 책임 |
| 왜 막혔나 | 차단·에스컬레이션 사유를 감사 원장에 기록 |
| 외부 모델이 본 것은 | 토큰화된 입력만(원본 PII는 국내 볼트 밖으로 안 나감) |

- **자동화는 신용 결정을 대체하지 않는다.** 준비·트리아지·근거 수집·승인 대기 초안만 자동화하고, 규제·고위험 결정은 사람이 소유한다.
- 모든 자동화는 무엇을 읽었는지·어떤 스킬을 썼는지·어떤 승인이 필요한지를 남긴다(프롬프트 설계 원칙).
