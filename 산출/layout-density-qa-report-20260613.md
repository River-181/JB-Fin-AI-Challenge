# UI 레이아웃 밀도 개선 QA 리포트

## 목표

JB LocalGuard OS의 주요 화면에서 콘텐츠가 아래로 밀리거나 카드 내부에 빈 공간이 과도하게 남는 문제를 줄이고, 금융 업무 콘솔처럼 상단 기준으로 촘촘하게 읽히는 레이아웃으로 정리한다.

## 확인한 문제

| 영역 | 문제 | 영향 |
| --- | --- | --- |
| 메인 워크스페이스 | `page-content`가 화면 높이를 강제로 채우면서 그리드 행이 늘어남 | 운영 목표 카드가 화면 하단으로 밀림 |
| 운영 목표 | 목표 달성률 카드가 실제 콘텐츠보다 큰 영역을 차지함 | 1920x1080 화면에서도 한눈에 보기 어려움 |
| KPI 카드 | 카드 내부가 `space-between`으로 분산 정렬됨 | 숫자와 설명 사이 간격이 커져 정보 밀도 저하 |
| 대시보드/업무 카드 | 일부 패널과 그리드에 남는 높이와 세로 분산 정렬이 유지됨 | 화면별 카드 높이가 불균형해 보임 |
| 칸반 보드 | 전체 화면 높이를 강하게 차지하는 `min-height` 사용 | 보드가 실제 카드 수보다 커 보임 |

## 적용한 개선

| 수정 항목 | 적용 내용 |
| --- | --- |
| 화면별 제어 | `#page-content`에 `view-dashboard`, `view-goals` 같은 현재 화면 클래스를 부여 |
| 상단 정렬 | 페이지, 패널, 그리드, 카드에 `align-content: start`, `align-items: start`, `align-self: start` 적용 |
| 강제 높이 완화 | `page-content`, `dashboard-grid`, `live-panel`, `work-item`의 과도한 높이 사용을 줄임 |
| 카드 밀도 개선 | KPI 카드와 업무 카드의 padding, gap, font scale을 조정 |
| 운영 목표 최적화 | `운영 목표` 화면의 목표 카드 5개가 상단에 모이도록 데스크톱 5열, 중간 해상도 3열, 태블릿 2열, 모바일 1열 그리드와 높이 제한을 검증 |
| 반응형 보정 | 1180px 이하 2열, 720px 이하 1열로 자연스럽게 축소 |

## Playwright 검증

새 테스트 `goals page keeps cards compact and top aligned`를 추가했다.

검증 기준:

- 1920x1080 뷰포트에서 운영 목표 헤더와 카드 사이 간격이 20px 이하
- 목표 달성률 패널 높이가 380px 이하
- 목표 카드 5개가 모두 렌더링
- 각 목표 카드 높이가 118px 이하
- 각 목표 카드가 상단 430px 이내에 배치
- 캡처 저장: `test-results/screenshots/goals-compact.png`

## 남은 제한사항

- 실제 운영 데이터가 많아질 경우 표/차트형 화면은 별도 가상 스크롤 또는 페이지네이션 검토가 필요하다.
- 모바일 390px 기준에서 오른쪽 상세 패널은 별도 하단 시트 UX까지 고도화하면 더 안정적이다.
- 현재 검증은 정적 MVP 기준이며, API 지연/실패 상태가 실제 서버와 연결되면 추가 레이아웃 QA가 필요하다.
