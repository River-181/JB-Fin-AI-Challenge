# 증빙 추적 체인 (Traceability Chains)

> 목적: 외부 출처가 어떻게 Pain Point가 되고, 어느 에이전트·스킬의 판단으로 이어지며, 승인 게이트를 거쳐 감사 로그에 남는지를 끝까지 추적한다. 심사 3.1·3.2(판단·행동·검증 구조와 흐름의 논리성), 4.1(문서 간 일관성), 5.5(설명가능성·책임소재)에 답한다.
> 단일 출처: [docs/_canon.md](../_canon.md) §2(에이전트 표시명)·§8(운영 계약). 수치 출처: [심사 인용 카드](01-심사인용카드.md) · [Pain Point 근거](../05_리서치/pain-point-evidence.md)
> 최종 갱신: 2026-06-14

## 운영 계약 (앱과 문서 공통)

```text
Case → AgentRun → Agent → Skill → Evidence → Approval → Audit
```

- 고객 대상 행동은 승인 전 자동 실행 금지. 고위험 fraud는 외부 접촉 차단.
- 승인 레벨: **L0**(관찰/로그) – **L1/L2**(RM 검토/승인) – **L3**(준법/법률) – **L4**(차단·상위검토).
- 모든 판단은 Evidence와 Audit으로 연결. 에이전트는 결론만이 아니라 **근거·불확실성·다음 확인 항목**을 남긴다.

추적 표준 체인:

```text
출처(기관 데이터·경보) → Pain Point → 에이전트/스킬 판단 → Approval(승인 게이트) → Audit(감사 로그)
```

## 예시 1 — 소상공인 상환위험 (히어로: 전주 중앙로 카페 · JBG-104)

| 단계 | 내용 |
| --- | --- |
| **출처** | 한국은행 「금융안정 상황」(2024말): 취약 자영업자 42.7만명(13.7%)·연체율 비은행 3.43%·취약 11.16% ([카드 A-2](01-심사인용카드.md)); 국세청: 2024년 폐업 100.8만명·폐업률 9.04% ([카드 A-3](01-심사인용카드.md)). 내부 이벤트: 운전자금 1.8억 노출·카드매출 둔화. |
| **Pain** | 금리 부담 + 매출 둔화로 인한 상환 스트레스 (riskScore 88, urgent). |
| **에이전트/스킬** | **상환위험 분류 에이전트**(케이스 owner)가 현금흐름·다중채무·금리민감도를 분류. **정책금융 매칭 에이전트**가 금리/상환 지원제도 후보 매칭. **포트폴리오 분석 에이전트**가 세그먼트 맥락 제공. → 결론·불확실성·다음 확인항목과 근거 링크를 Evidence로 생성. |
| **Approval** | 고객 대상 행동(RM 콜백·상환지원 안내 초안)은 **L1/L2 — RM 최종 승인자** 승인 전 발송 금지. 케이스 상태 `Approval Pending`. |
| **Audit** | 사용한 출처·스킬·판단 근거·승인자·결정 시각을 감사 로그에 기록 → Evidence traceability 100%·Approval safety 100%. |

## 예시 2 — 전세사기 위험 (전세 보호 라인)

| 단계 | 내용 |
| --- | --- |
| **출처** | 국토교통부 전세사기피해지원위원회: 피해자 누적 3.9만건(2026.6) ([카드 B-1](01-심사인용카드.md)); HUG: 사고액 4.49조(2024 피크), 회수율 8~14%, 2025년 보증 강화 후 73.1% 감소 ([카드 B-2](01-심사인용카드.md)). 케이스 데이터: 임대차/전세대출 단계 전세가율·권리관계 신호. |
| **Pain** | 깡통전세·고위험 전세로 인한 임차인 보증금 손실 위험과 후속 대출 부실. |
| **에이전트/스킬** | **전세위험 관리 리드**가 라인을 지휘. **전세가율 분석 에이전트**(전세가율), **등기 권리 분석 에이전트**(권리관계), **임차인 손실위험 에이전트**(손실 시나리오)가 신호를 산출하고, **계약 체크리스트 에이전트**가 보증보험·은행 연계 체크리스트를 연결. → 결론이 아닌 "검토 필요 신호"로 Evidence 생성. |
| **Approval** | 임차인 안내·보증 점검 초안은 **L1/L2 — RM 최종 승인자** 승인 후에만 발송(승인형 발송). 권리/보증 쟁점이 크면 **L3 — 준법 최종 승인자**로 에스컬레이션. |
| **Audit** | 전세가율/권리관계/보증보험/은행 연계 체크리스트 연결 100%(Jeonse safe-contract KPI), 출처·판단·승인 기록을 감사 로그에 남김. |

데모: `?demo=jeonse`.
※ 등기·실거래 데이터는 자동 대량수집이 제한·유료이므로 케이스 단위 소량 조회로만 사용 (상세: [데이터·API·라이선스 인벤토리](../05_리서치/data-api-license-inventory.md) §A·D).

## 예시 3 — 보이스피싱·이상거래 (이상거래 라인)

| 단계 | 내용 |
| --- | --- |
| **출처** | 금융위원회: 보이스피싱 피해액 2025년 1~11월 1.13조(+56.1%) ([카드 C-1](01-심사인용카드.md)); 경찰청: 50대 이상 53%·기관사칭형 51%·1건 평균 5,301만원. 외부 경보: ASAP 정보공유 플랫폼 ([카드 C-2](01-심사인용카드.md)). 내부 이벤트: 이상거래 신호. |
| **Pain** | 고령(50대 이상) 고객에 집중되는 보이스피싱 피해 + 단독 FDS의 탐지 한계. |
| **에이전트/스킬** | **이상거래 탐지·차단 에이전트**가 외부 경보(ASAP 등)와 내부 케이스를 결합해 이상거래를 탐지. **준법 검토 에이전트**가 차단/지연의 적법성·근거를 검토. → 근거·불확실성을 Evidence로 생성. |
| **Approval** | 고위험 fraud는 고객 대상 행동 자동 실행 금지 + **외부 접촉 차단**. 차단·상위검토는 **L4**, 준법 검토는 **L3 — 준법 최종 승인자**. |
| **Audit** | 외부 경보 출처·탐지 근거·차단 결정·승인자를 감사 로그에 기록 → 책임소재·설명가능성 확보(심사 5.5). |

데모: `?demo=phishing`.

## 체인 요약

| 예시 | 출처 | Pain | 에이전트 | 승인 레벨 | KPI 연결 |
| --- | --- | --- | --- | --- | --- |
| 1 소상공인 | 한은·국세청 | 금리 부담+매출 둔화 | 상환위험 분류 에이전트 | L1/L2 (RM) | Triage time 50%↓, Evidence 100% |
| 2 전세 | 국토부·HUG | 깡통전세·보증금 손실 | 전세위험 관리 리드 | L1/L2, 쟁점 시 L3 | Jeonse safe-contract 100% |
| 3 보이스피싱 | 금융위·경찰청·ASAP | 고령 피해·FDS 한계 | 이상거래 탐지·차단 에이전트 | L3/L4 (차단·준법) | Approval safety 100%, 사후관리 누락 0건 |

설계 원칙: 고위험 케이스는 자동 실행이 아닌 **escalation·승인 중심**으로 처리하고, 위험 판단 근거·사용 스킬·출처를 모두 Audit Log에 남긴다.
