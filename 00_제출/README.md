# 05_제출

대회 제출용 산출물을 정리하는 공간입니다.

## 이 폴더의 역할

이 폴더는 심사 제출 직전에 확인할 패키지 인덱스입니다. 실제 원문 문서는 `docs/04_submission`과 `산출`에 있고, 이 폴더는 제출물 구성과 검증 순서를 정리합니다.

## 제출 패키지 구성

- MVP 제안서
- 평가항목 대응표
- 앱 데모
- 근거 문서
- Agent/Skill 설계 문서
- 아키텍처 다이어그램
- 검증 스크립트 결과

## 심사용 확인 순서

1. 제안서에서 문제, 솔루션, JB 사업 연결성을 확인합니다.
2. 앱 데모에서 케이스 선택, AgentRun, 승인, 감사 로그를 직접 확인합니다.
3. Agent 시스템 문서에서 복수 Agent와 스킬 장착 구조를 확인합니다.
4. 아키텍처 문서에서 시스템/데이터/API/사용자 흐름을 확인합니다.
5. 증빙 문서에서 pain point와 공식자료 출처를 확인합니다.
