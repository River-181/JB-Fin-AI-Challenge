## 1. NVIDIA SkillSpector — 에이전트 스킬 보안 스캐너 ★즉시

**무엇**: AI 에이전트 스킬(Claude Code·Codex CLI·Gemini CLI)을 **설치 전** 정적·LLM 2단계로 스캔해 0~100 위험점수를 매기는 오픈소스 CLI. NVIDIA 공식, Python.
- 레포: `https://github.com/NVIDIA/SkillSpector` (Apache-2.0, v2.0.0, ~5.5k★, mid-2026 활성)
- 탐지: **68개 패턴 / 17개 카테고리** — prompt injection · data exfiltration · privilege escalation · **supply chain** · excessive agency · system prompt leakage · memory poisoning · tool misuse · MCP tool poisoning 등. AST 위험호출(exec/eval/subprocess)·taint tracking·YARA 시그니처 포함.
- 점수: 0–20 LOW(SAFE) / 21–50 MEDIUM(주의) / 51–80 HIGH(설치금지) / 81–100 CRITICAL(설치금지). 실행 스크립트 포함 시 ×1.3.

**왜 우리에게 즉시 필요한가**: 우리 하네스는 **서드파티 스킬·플러그인을 깔고 팀에 공유**한다(taste-skill·notebooklm·impeccable·ui-ux-pro-max 등 외부 GitHub). NVIDIA 조사: 스킬의 **26.1%가 취약점, 5.2%가 악성**. 스킬=에이전트 권한으로 실행되는 텍스트 명령 → 공급망 리스크. 금융 공모전(PII 비반출이 핵심 가치)에서 "우리가 쓰는 도구조차 검증한다"는 **메타 거버넌스 자산**.

**설치/사용** (검증: 공식 README):
```sh
uv tool install git+https://github.com/NVIDIA/skillspector.git   # CLI 설치
skillspector scan ./SKILL.md                # 단일 스킬
skillspector scan https://github.com/user/x # 원격 레포
skillspector scan ./my-skill/ --no-llm      # 정적만(빠름, API키 불필요)
```
LLM 모드 provider: `anthropic` / `claude_cli` / `codex_cli` 등 지원 → 우리 환경에서 그대로.

**⚠️ 한계(공식 명시)**: **"Non-English content: May miss patterns in other languages"** — 우리 스킬·문서는 대부분 한국어 → **정적 모드만으론 한국어 페이로드 누락 가능**. 대응: 우리 자체 스킬에 대해 **LLM 모드(`--no-llm` 끄고 anthropic/claude_cli)** 병행. 이미지 내 텍스트·런타임 동작은 스캔 불가(정적 한정). 클린 리포트 = "안전 확정"이 아니라 "명백한 게 안 걸림".

**적용 권고**: `bootstrap.sh` STEP에 "서드파티 스킬 설치 전 `skillspector scan` 게이트" 추가 + [[registry-cli]] 등록. 기존 설치 스킬(skills-lock.json) 1회 베이스라인 스캔으로 점수 감각 보정.

## 2. humanize-korean (im-not-ai) — 한국어 AI티 윤문 플러그인 ★공식화

**무엇**: AI(ChatGPT·Claude·Gemini)가 쓴 한국어의 "AI 티"(번역투·과잉 영어인용·기계적 병렬·피동남용·서식남용)를 **내용 불변**으로 윤문하는 Claude Code 플러그인. 10대 카테고리×40+ 서브패턴, S1/S2/S3 심각도, Fast(단일콜)·Strict(5에이전트) 2모드.
- 레포: `https://github.com/epoko77-ai/im-not-ai` (MIT, Python 90%)
- **현황**: 이 스킬은 **이미 세션에 로드되어 있음**(`humanize-korean` + 서브에이전트 `ai-tell-detector`·`korean-style-rewriter`·`content-fidelity-auditor` 등). 즉 **개인 전역엔 있으나 팀 프로젝트 세트엔 미등록** → 이번에 **마켓플레이스로 공식화**.

**왜**: 본선 산출물(제안서·명세서·발표 스크립트)이 LLM 초안 기반 → 심사 제출물의 "AI 티 제거"는 가독성·완성도(심사 4.5)와 직결. 영어권 휴머나이저(QuillBot 등)는 한국어 번역투를 못 잡음.

**설치/사용** (검증: 공식 README):
```sh
/plugin marketplace add epoko77-ai/im-not-ai
/plugin install humanize-korean@im-not-ai
# 호출: "이 글 AI 티 없애줘" / /humanize
```

**적용 권고**: `.claude/settings.json`의 `enabledPlugins` + `extraKnownMarketplaces`(`github:epoko77-ai/im-not-ai`)에 추가 → 팀 자동 동기화. ⚠️ settings.json은 **git 추적·팀 공유 SSOT**라 enabledPlugins 편집 시 [[registry-plugins]] 자동 재생성(plugin-inventory.mjs). detection-bypass가 아니라 **품질 개선** 도구임을 발표 윤리상 명확히.

## 3. Ollama + 한국어 로컬모델 — 시연 차별점 ★즉시

**왜 최우선**: 6/30 확정 시연원칙 ①"**로컬 모델 실동작**(은행 보안·망분리·PII 비반출)". 이게 제품의 핵심 해자를 라이브로 증명하는 도구.

**런타임 = Ollama** (맥 Apple Silicon, `ollama run`으로 망분리 환경 즉시 시연. LM Studio는 GUI 보조). 모델은 **라이선스↔내러티브 트레이드오프**:

| 모델 | 크기 | Ollama | 라이선스 | 본선 적합 |
|------|------|--------|---------|----------|
| **LG EXAONE 3.5** | 2.4B/**7.8B**/32B | `ollama pull exaone3.5` | ⚠️ 비상업(연구) — **데모 OK** | 🥇 국산 주권모델 = JB(국내 금융)·망분리 내러티브 최적 |
| **Qwen2.5 / Qwen3** | 7B | `ollama pull qwen2.5` | ✅ Apache-2.0(상업) | 🥈 상업 라이선스 안전판, 7~8B 한국어 강함 |
| SKT A.X 4.0 Light | 7B | HF→gguf | 비교적 관대 | 국산+상업 대안 |

**권고**: 히어로 시연=**EXAONE 3.5 7.8B**(국산·망분리 메시지 일치), 제품화 슬라이드엔 **Qwen 계열을 상업 라이선스 백업**으로 병기. ⚠️ EXAONE 비상업 제약 → "데모용" 명시 + 발표 전 라이선스 원문 1회 재확인.

- **Cytoscape.js** (조직도 메인UI): MIT, **`<script>` 태그 로딩** = 현 바닐라JS MVP에 번들러 없이 통합. 노드클릭·줌/팬·계층확장 기본. (GoJS·balkangraph=상용 → 탈락.)
