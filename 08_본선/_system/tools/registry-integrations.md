# registry-integrations

> 팀이 실제로 쓰는 **협업·공유·동기화 도구** 레지스트리.
> MCP로 연동되는 도구(GitHub·Google Drive·Notion)의 와이어링·인증은 [[registry-mcp]]에서 관리하고, 여기서는 **무엇을 어디서 공유·보고하는지(협업 역할)**에 집중한다.
> 팀별 구체정보(Discord 초대링크·Syncthing folder ID·Drive 폴더 ID)는 보안상 비워둔다 — `_(팀 확정)_`.

---

## 협업 도구 레지스트리

| 도구 | 용도 | 무엇을 여기서 하나 (공유·보고 역할) | 동기화·접속 방법 | 대외비 경계 | 상태 |
|------|------|--------------------------------|----------------|-----------|------|
| **Discord** | 팀 실시간 커뮤니케이션·알림 | 잡담·의사결정 토론·진행 알림 + **회의 녹음(Craig 봇, 멀티트랙)**·채널별 주제 소통 | 서버 초대 링크로 합류 → 채널 구조 `_(팀 확정)_` | 대외 공개 금지 채널 분리(고객 PII·.eml 첨부 금지) | 활성(사용 중) |
| **Syncthing** | 볼트 P2P 로컬 동기화 | 옵시디언 볼트를 팀원 기기 간 직접 동기화(중앙 서버 없음, 빠른 **실시간 파일·지식 공유**) | folder ID 공유 → Syncthing GUI에서 폴더 수락. `.stfolder/`·`.stignore` gitignore(line 35-36) | 동기화 폴더 자체가 대외비 — 외부 기기·클라우드 미러 금지 | 활성(사용 중) |
| **GitHub** | 코드·문서 SSOT·**결과물** 공유 | 소스·문서 버전관리·PR·이슈, 6/29 후 오픈소스 공개(결과물 위주) | fork = `River-181`. 와이어링/인증 → [[registry-mcp]]·[[registry-cli]] | ⚠️ **6/29 전 비공개(private)** — 대외비. 공개 전환은 사람만 승인 | 활성(비공개) |
| **Google Drive** | 대용량·바이너리 + **회의 녹음 보관** | 시연영상·대용량 산출물 + **회의 녹음(WAV) 원본 보관** 등 git에 안 올리는 파일 | MCP 연동 → [[registry-mcp]]. 폴더 ID `_(팀 확정)_` | ⚠️ `.webm`/`.mp4`/`.eml`/녹음/PII 공개 검색노출 금지 — 팀 내부 폴더 한정 | 활성 |
| **Notion** | **내부 미팅 페이지** + 외부 게시 | **회의 타임라인·녹음 정리(비번 보호 페이지)** + 외부 공유용 문서 게시(볼트 → Notion) | `notion-publish` 스킬. MCP 연동 → [[registry-mcp]] | 게시 전 PII·대외비 검수. 6/29 전 외부 게시는 사람만 승인 | 활성 |

## 협업 채널 가이드 — 무엇을 어디서

| 무엇을 | 어디서 |
|-------|-------|
| 코드·문서(SSOT) | **GitHub** (PR·이슈·버전관리) |
| 볼트 동기화 | **Syncthing** (기기 간 P2P) |
| 잡담·알림·의사결정 | **Discord** |
| 대용량·바이너리(시연영상 등) | **Google Drive** |
| 외부용 게시 | **Notion** (`notion-publish` 스킬) |

**대외비 경계 (전 채널 공통):**
- **6/29 공식발표 전**: GitHub 비공개 유지, 외부 게시 금지.
- **`.eml`(원본 메일)·PII**: 어떤 채널에도 업로드 금지(GitHub·Drive·Notion·Discord 첨부 포함).
- **시연영상(`.webm`/`.mp4`)**: 공개 검색노출 금지 — 대회 공식 채널 또는 팀 내부 Drive로만.
- 외부 제출·공개·고객 대상 행위는 **사람만 승인**(AGENTS.md 결정모델).
