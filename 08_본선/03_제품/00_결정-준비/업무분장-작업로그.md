# 업무분장 반영 + 자율 작업 로그 (2026-07-03)

| 담당 | 역할 |

| **이승보** | 프로토타입 공유(zip/GitHub repo)·설계도·**기존 은행 DB 연결 방법 명문화** |

| 공통 | 미해소 문제·궁금·수정할 것 바로 남기기 + 대회 필요한 것 알잘딱 |

## 내가 지금 자율로 진행 (김주용 레인 + 크로스커팅)
**Workflow `wtxnqwxss`** (Codex 위임·리서치 기반, 백그라운드) — 4개 설계 문서 초안+심사검증, 산출 `00_결정-준비/설계/`:
1. **에이전트 아키텍처 재설계** — 사용자 지적("5개로는 부족") 반영. 뽀족한 전문가 에이전트 로스터(직군×도메인, 각=지식+규정+메모리+툴+스킬, 서버 영속, Claude/Codex/로컬 런타임). "몇 개가 맞고 왜"를 리서치(D8/D9/D14/D15+canon 14)로.
2. **"하나의 금융 AX 콘솔" 서사** — 3기능 합이 아닌 하나의 운영체계 논거(운영계약·PII·역할축·모듈식).
3. **은행 자동화 vs 우리 차별점** — 영욱님 GPT 딥리서치 전, 기존 리서치로 초안 + 영욱님 확인 필요 지점 플래그.
4. **스킬/플러그인/외부 플러그인·데이터 구상** — 에이전트별 툴/스킬/MCP·외부 오픈API 카탈로그.

## 팀원 레인 — 대기(내가 침범 안 함, 스캐폴딩·질문만)
- **승보**: 프로토타입 공유(오면 일괄 진행)·DB 연결 명문화. → 내 #4가 참고 자료 됨.
- **민주**: 디자인·발표·이름/슬로건/승인UX. → [[Q11-시스템 이름·브랜딩]]·[[Q12-위험 업무 승인 UX 구체 스펙]]에 후보 정리돼 있음.
- **영욱**: 공공데이터·KIPRIS·전세/피싱 출처·은행 자동화 차별점. → 내 #3·#4가 초안·플래그 제공.

## 업무분장에 없지만 챙길 것 (내 판단)
- **기능명세서 변경이력**(canon: 미기재=실격) — 제출물 필수.
- 제출물 공식 형식(MVP 제안서 7섹션·기능명세서 6파트, canon §6) 준수.
- 발표 예선 5분/결선 15분 런시트 + 리허설.
- 시연영상 ≤5분·≤100MB·비공개 업로드.

## ❓ 질문 (돌아오시면 답 부탁 — 그전까진 기본값으로 진행)
1. **에이전트 수**: "5개 부족" 동의. 리서치 결과 나오면 적정 규모 권고안 제시 예정(예상: 코어 오케스트레이터 + 직군별 전문가 + 통제군). **몇 개 상한선/선호 있으신가요?** (없으면 리서치가 권고하는 대로.)
2. **git 브랜치**: 지금 `ui-density-improvements`에 병렬 세션들과 공존 커밋 중. 결정-준비/설계 작업을 **별도 브랜치로 분리할까요?** (단 같은 워킹트리라 브랜치 전환은 병렬 세션에 영향 → 사용자 결정 필요. 기본값: 현 브랜치 유지, 내 파일만 `git commit -- <경로>`.)
3. **로컬모델 실동작**: 어느 머신(승보님 PC?)·어느 모델(EXAONE 3.5 / Qwen2.5 / Gemma)을 실동작 타깃으로? (기본값: 승보님 확보한 것 우선.)
4. **도메인 조합**: [[키스톤-확정]]의 하이브리드(옵션3) 권고 — 승보 프로토타입+팀 확정 대기.
5. **승보 프로토타입 공유 방식**(zip/GitHub) 받으면 그 구조에 맞춰 앱설계·기능명세서 일괄 정합.
