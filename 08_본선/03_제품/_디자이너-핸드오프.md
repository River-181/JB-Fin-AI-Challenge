# 디자이너 핸드오프 — 'JB금융-디자인' 세션용

> **대상**: 클로드 코드 세션 `JB금융-디자인`. **작성**: 문서/오케스트레이션 세션(River 팀장 지시 2026-07-04).
> **한 줄**: 문서세트 22종이 커밋(`a27ad9f`)됐다 — 이제 **뷰·컴포넌트·정보레이어·화면구성 확정 설계를 시작해도 된다.** 아래는 그 근거가 될 SSOT 문서 지도.

## 역할 경계 (핑퐁)
- **문서 = 문서 세션 소유**(정합·근거·SSOT). 디자인 파일 무접촉.
- **디자인 토큰·컴포넌트·뷰·정보구조 = 디자인 세션 소유**. 문서 세션 무접촉.
- **핑퐁**: 디자이너가 뷰/컴포넌트 초안 → 문서 세션이 문서 정합(용어·상태·권한·플로우 일치) 리뷰 → 반복.

## 디자인 정합 기준
- **토큰·컴포넌트 = MAIN 소스(사용자 확정 2026-07-04)**: 이승보 **JB_project2 `_vendor/JB_project2/app/styles.css`**(8,696줄·**118 CSS 토큰** `--color-brand-primary/accent/deep/navy`·`--color-console-nav/panel/app`·`--focus-green` 등 + **481 컴포넌트 클래스**, Pretendard Variable, radius 20/22/24). = **메인 디자인 토큰/컴포넌트 소스.** 여기 **없거나 부족한 것만** [[_JBFG-디자인-레퍼런스]](JBFG 브랜드 토큰·`token-data.json`·`screens/`)에서 **편입**. paperclip 비주얼/아이콘 사용 금지(구조만).
- UX: Enter-first 단순화 + **승인 UX 접근성**(hover만 아니라 focus/open/active 명시).
- 차별성 반영: [[차별성-경험레이어-서사]] — "AI는 판단할 여유를 만든다", 역할 기반 게이트.

> 김민주 회의 마무리 발언 "**8분 안에 피그마로 만들어 보내드릴게요**" = 디자이너 실작업 착수 신호. 아래는 그 첫 화면 근거.
- **출처 SSOT**: [[2026-07-04-전략회의2-메인대시보드-정리본]] + 반영 스펙 [[06-정보체계-뷰-데이터바인딩-스펙]] **§3.16 델타**(FR-16~27) + [[기능요구사항-7-4회의]] §3.
- **핵심 확정**: (1) **메인=업무 보드**(진입 즉시 오늘 할 일·진행 전/중/후 카운트·우선순위) (2) 케이스 드릴인 2계층(위=케이스/아래=에이전트·API) (3) **키보드 전용·마우스 금지** 밀어내기(엔터 승인→스트로크→딤드→다음, 숫자키 열기, 엔터 2단확인) (4) LLM **비동기+알림(띠링)**·완료=어두운 색 (5) **결과물 MD 뷰**(통합본+개별, 플로팅 모달, 옵시디언식 `[[개별MD]]`·`@에이전트`→S-07-detail, 스킬·출처 하단) (6) 딤드/블러+화살표 스크롤 인식 (7) **2계열사만** 시연.
