# 오케스트레이터

> 역엔지니어링/브레인스토밍으로 채울 예정

---

## 오케스트레이터 역할

> 서브에이전트들의 실행 순서·병렬화·에러 핸들링·결과 통합을 담당.

---

## 씨앗 포인트

- **씨앗**: FSM 5단 상태 모델 — New → 진행중 → 검토 → (승인 대기) → 완료 / 차단
- **씨앗**: 케이스 상세에서 인라인 라이브런 스트리밍 — 실시간 에이전트 실행 과정 표시
- **씨앗**: 고객 대상 행동은 Approval 게이트를 통과해야만 실행됨

---

## 실행 플로우

> 작성 예정

```
위험 신호 입력
  → Triage Agent (분류)
  → Risk Assessor (위험도 평가)
  → Draft Writer (행동 초안 작성)
  → Compliance Checker (규정 검증)
  → [승인 게이트] ← 여기서 사람이 개입
  → (승인 시) 행동 실행
  → AuditEvent 기록
```
