# P0 정의 합의

## ① 목표

정의서·11블록을 확정하고 전북/광주, 배포형/로컬형, DB 범위 등 미결 결정을 옵션 병기 상태에서 합의한다.

## ② 세부 작업

- [ ] 제품 한 줄 정의와 코어 베팅(PII 비반출×승인우선×멀티도메인)을 정의서에 고정 #agent
- [ ] 11블록 캔버스의 `[확정]`·`[조건부]`·`[격리]` 마커를 재검토 #demo
- [ ] 히어로 은행 옵션A(전북)／옵션B(광주) 결정안 작성 #ui
- [ ] 배포형／로컬형 선택 기준과 본선 시연용 범위 분리 #security
- [ ] DB 범위 옵션A(정적/로컬 상태)／옵션B(서버 API+영속 DB)／옵션C(하이브리드) 비교 #data

## ⑤ 산출물

- 확정 대기 정의서 체크리스트

## ⑥ 리스크·미결

- 전북/광주는 회차 확정 전까지 옵션A/옵션B로 병기한다.
- 배포형/로컬형은 보안·시연 완주 가능성·설치 난이도 기준으로 비교하되 단정하지 않는다.
- DB 범위는 본선 시간과 검증 가능성에 따라 정적/로컬, 서버 API+영속 DB, 하이브리드 중 선택한다.
- 격리 수치(RM 업무량, ROI, 시장규모 등)는 본문 확정값으로 쓰지 않는다.
