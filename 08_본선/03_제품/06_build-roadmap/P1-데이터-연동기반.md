# P1 데이터 연동기반

## ① 목표

공공/내부 데이터, API, 라이선스, 코어뱅킹 연동 경계를 설계하고 본선 시연에서 안전하게 재현할 데이터 계약을 정한다.

## ② 세부 작업

- [ ] 공공 데이터 후보를 즉시 사용 가능/제휴 필요/확인 필요로 분류 #data
- [ ] 내부 정보계 Read-only CDC, 토큰화 프록시, 승인 후 MCI/EAI Write 경로를 분리 #data
- [ ] 코어뱅킹 직접 쓰기 제외 원칙과 승인 후 연계 옵션을 명세 #security
- [ ] 상용 CB·카드매출·사기방지 데이터의 재배포·반출 제약을 표로 정리 #data
- [ ] 등기정보광장·보증기관·금융결제원 한도/약관은 확인 과제로 격리 #data
- [ ] 본선용 샘플 데이터와 실제 연동 후보를 분리해 시연 fixture 설계 #demo

- 내부 데이터는 식별형 내부경영관리와 가명분석 레인을 분리하는 옵션으로 설계한다.
