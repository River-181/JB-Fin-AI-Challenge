# 설계도 인덱스 — 개발 와이어프레임 빌드용 청사진

> **목적**: 기획 문서(정보체계·플로우·기능명세)를 **"설계도만 봐도 개발 와이어프레임(Figma/HTML)을 짤 수 있는"** 화면 단위 청사진으로 승격한다. 각 설계도는 7단 구조로 통일 — **목적·진입 → ASCII 와이어프레임 → 컴포넌트 트리 → 데이터 바인딩 → 상태 → 키보드·인터랙션 맵 → FR 근거**.
> **근거 SSOT**: [[06-정보체계-뷰-데이터바인딩-스펙]](뷰 카탈로그 S-00~S-17·ViewModel·컴포넌트명) + [[2026-07-04-전략회의2-메인대시보드-정리본]](메인 대시보드 확정) + [[기능요구사항-7-4회의]](FR-01~27). 디자인 토큰=`_vendor/JB_project2/app/styles.css`(MAIN).
> **참조 피델리티**: 팀이 그린 실제 Figma 메인 대시보드 프레임(망상궤도 파일, node 4924-844) — `01-메인-업무보드`가 그 빌드 청사진이다. 디자이너 세션(`JB금융-디자인`, 김민주)이 이 설계도를 1차 근거로 Figma 구현.

## 쇼케이스 빌드플랜 (도장깨기)

- [[_쇼케이스-빌드플랜]] — **풀 쇼케이스 10흐름 × 6페르소나** 빌드 계획. 최신 히어로=Figma 5페르소나(김건우(CCL 히어로 유지)+오태림(JBWC)·문서희·윤복례·임세빈·전세, 2계열사). 실 LLM=F1(김건우 CCL-0001), 실 API=F2(기상)·F6(전세). 도장깨기 3라운드(코어→차별성→풀) + Gate 0(`/llm` 이식).

## 화면목록 (파리티 뷰맵)

- [[_화면목록-뷰맵]] — **계열사×역할 양축 전체 화면 목록**. JB_project2 실측 **98뷰**(CCL14·FDR15·JPO29·JBWC24·base15·rm1)를 계획 화면(S-ID/설계도)에 매핑. 커버리지: **66/98 제네릭 템플릿 커버 · 29 도메인 특화 명세 필요**(FDR5·JPO15·JBWC9) · 3 부분특화. 데모 크리티컬 **8뷰**. → 설계도 5종은 CCL/제네릭 경로의 빌드 청사진, 특화 뷰는 이 뷰맵이 백로그.

## 실측 정합 주의 (구현현황 대비)

설계도는 **목표 화면**을 그린다. 코드 실측([[구현현황-JB_project2]])과 다음이 다르니 빌드 시 유의:
- **감사 해시체인**: 현재 base 앱만 체인(FNV-1a), 4개 역할 콘솔은 flat 로그 → 05 설계도의 "전 콘솔 해시검증"은 목표(미구현).
- **실 LLM 호출 없음**: 40개 에이전트 출력=템플릿 문자열, 실 호출은 전세 `?live=1`(data.go.kr) 2건뿐 → 03 설계도의 실행은 파이프라인 실재+LLM 텍스트 모의.
- **케이스=JS객체/localStorage**(5 DB), 마크다운 파일 아님 → 02 설계도의 결과물 MD 뷰는 렌더 표현(FR-08 파일화는 목표).
