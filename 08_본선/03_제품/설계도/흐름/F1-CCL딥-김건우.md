# F1 CCL 기업여신 딥(실 LLM)·히어로 — 김건우(JBG-104/CCL-0001) 흐름

> **[대외비]** 인물·거래는 전부 합성(PII無, [[JB-콘솔-페르소나-로스터]] JBG-104 원본 바이오). 외부 공개 전 사람 최종승인 필요. 본 문서는 화면 자체를 다시 그리지 않고 [[01-메인-업무보드]]·[[02-케이스-상세]]·[[03-에이전트-실행뷰]]·[[04-승인-대기·결정]]·[[05-통계-추적-패널]]의 S-ID를 그대로 참조한다. 이 문서가 새로 정의하는 것은 **김건우 케이스 하나가 그 화면들을 어떤 순서·데이터·실동작 등급으로 관통하는가**다.

> **히어로 확정(2026-07-04)**: 딥 실LLM 히어로 = **김건우(전주 중앙로 카페, CCL-0001)**. 유지(폐기 아님) — 쇼케이스는 6페르소나로 확장하지만, 딥 실LLM 흐름의 주인공은 처음부터 끝까지 김건우다. 오태림(JBG-205)은 별도 [[F3-JBWC-기업여신-딥-오태림|F3 JBWC 흐름]](실 LLM 2번째 후보 or 확장성 시연)이며 딥 히어로가 아니다.

## 1. 한 줄 시나리오 + 페르소나

**김건우, 전북 전주(중앙로) 카페 사장, 개인사업자 운전자금.** 전북은행 CCL 기업여신 케이스, Case ID `CCL-0001`(캐논 JBG-104), Risk 88, 상태 승인대기, **히어로 케이스(8단계 타임라인)**. 카드매출은 둔화되는데 운전자금 1.8억의 금리 부담은 그대로여서, 대환·정책금융 검토와 상환 스트레스 점검이 동시에 필요해진 케이스다. (원문: [[JB-콘솔-페르소나-로스터#JBG-104 — 김건우]])

**왜 이 케이스가 딥 실 LLM 히어로인가**
- **PR#1이 이미 여기에 기배선돼 있다.** `feat/live-agent-ccl-financial`(River-181 fork, `LSB-afk/JB_project2#1`)이 `CCL-0001` 케이스를 대상으로 실 LLM 슬라이스(claude/codex 라우팅 + `harnessGuardCheckPII`/Assertions 재검)를 이미 구축해 놓았다 — **재시드 없이 최단 경로**로 실 LLM을 보여줄 수 있는 유일한 케이스다.
- **텍스트 생성 가치가 가장 선명하다.** 위험신호 조기감지·상환위험 분류는 규칙 기반 파이프라인으로도 흉내 낼 수 있지만, 콜백 문안·품의/회신 초안을 실제로 "쓰는" 작업은 LLM이 아니면 흉내만 난다.
- **Zero-PII 최소필드 + 로컬 PII 처리 하이브리드를 가장 정직하게 보여준다.** 카페 사장 개인사업자 정보(성명·사업자번호·계좌)는 로컬모델이 처리하고, 프런티어(claude/codex)에는 최소필드만 나간다.
- **검증 에이전트가 자기 오류를 잡는 장면이 신뢰의 절정이다.** 준법 검토 에이전트가 초안의 인용 오류(금리 조건·정책자금 요건 등)를 스스로 재검해 되돌리는 장면이 F1의 클라이맥스다.

## 3. 실 LLM 포인트 상세 — 모델 선택 게이트웨이(claude/codex/로컬)

**어디서 실제로 도는가**: 스텝 5(품의/회신 초안 생성)가 유일한 실 LLM 지점이다. PR#1(`feat/live-agent-ccl-financial`)이 이미 `CCL-0001`에 배선한 `/llm` 게이트웨이(`api-proxy.mjs` `handleLlm`)가 claude↔codex CLI를 라우팅하며, 결정 4·6(2026-07-04)에 따라 **로컬모델(EXAONE·Qwen 등)까지 어드민이 선택**하는 모델 선택기로 확장한다.

**PII 하이브리드(결정 6)**
- **로컬모델이 PII 포함 처리** — 원문(성명·사업자등록번호·계좌) 반출 없이 온프레미스에서 처리.
- **프런티어(claude/codex)는 비민감·최소필드만** — Zero-PII 페이로드 예시(합성):

```json
{
  "caseId": "CCL-0001",
  "riskLevel": "high",
  "affiliate": "jeonbuk-bank",
  "sector": "카페(개인사업자)",
  "workingCapitalBand": "1.5억~2억",
  "cardSalesTrend": "둔화",
  "rateBurden": "high",
  "policyMatchCandidates": ["대환대출", "소상공인 정책금융"]
}
```
상호명·대표 성명·사업자등록번호·계좌번호는 이 페이로드에 없다 — `harnessGuardCheckPII()`가 반출 직전 이 계약을 강제한다.
- **로컬 불가 시 codex 폴백**(정직 표기) — 로컬모델 인프라 미비 시 claude/codex 프런티어로만 진행하고, 그 사실을 데모 대사에서 숨기지 않는다.

**폴백 사다리** — [[Q14-오류로깅-폴백사다리|Q14]] 재현: 정상 호출(claude 또는 로컬) → 강제 실패 시 반대 엔진으로 1회 폴백 → 둘 다 실패 시 `escalated: true, queue: "human"` → [[04-승인-대기·결정]] 승인 큐로 격상. 모든 시도가 `llm-runs.jsonl`에 기록된다(`runId, engine, tier, errorClass, retry, fallbackPath`).

**원가 근거**: 실측 1회 호출 in 49K(대부분 캐시)·out 3토큰·$0.115, break-even 대비 10배 여유([[Q13-토큰비용-유닛이코노믹스|Q13]]).

**검증 에이전트가 자기 오류를 잡는 장면** — 품의초안이 금리 조건이나 정책자금 요건을 잘못 인용하면, 준법 검토 에이전트가 `PolicyCitationBox`/근거 대조에서 불일치를 발견해 `AgentRunCard`를 `needsReview`로 되돌린다. [[03-에이전트-실행뷰]]의 `Shift+Enter`(반려/재검토, 사유 필수) 경로를 그대로 탄다.

## 4. 가드레일·승인·감사

- **PII 반출 차단**: `harnessGuardCheckPII()`(`harnessCore.js:74-110`, 정규식 기반 주민번호/전화번호/11자리 이상 숫자열 탐지) — `beforeCaseCreate`/`afterAgentRun`/`beforeCustomerMessage` 훅에서 강제. 위반 시 값이 아니라 "어떤 종류가 걸렸는지"만 표시(원문 미노출).
- **사람 최종승인**: [[04-승인-대기·결정]] `S-04-detail` 3분할 — 콜백 문구·대환 조건이 초안과 다르게 조정될 여지가 있으므로 승인/거부/수정후승인 전 경로를 모두 지원.
- **감사 봉인 — 정직 표기**: 감사 이벤트 자체(구조화 로그)는 JB_project2에도 실재한다. 다만 **해시체인 무결성 검증 UI(`prevHash→hash` 검증, `verifyAuditChain()`, `exportAuditJson()`)는 예선 MVP(`02_제품/app/`) 한정 구현**이며 JB_project2(제출 정본)에는 아직 이식되지 않았다 — F1 시연에서 감사 탭을 열 때 "해시체인 검증"까지 보여주려면 [[05-통계-추적-패널]]의 예선 MVP 화면을 별도로 인용해야 한다. 과장 금지.

## 7. 의존성

- **Gate 0 — 모델 선택 게이트웨이 JB_project2 이식**: 현재 `handleLlm`/`llm-runs.jsonl`/`liveLlmBlock`은 예선 `02_제품`에만 있다. F1의 실 LLM 지점(스텝 5)은 이 게이트웨이가 JB_project2(제출 정본)에 이식돼야 성립하며, claude/codex 2웨이 라우팅을 **로컬모델 포함 3웨이**로 확장해야 한다 — 미이식 상태에서는 F1 전체가 🟡로 강등된다.
- **PR#1 머지**: `feat/live-agent-ccl-financial`(River-181 fork, `LSB-afk/JB_project2#1`)이 이미 `CCL-0001`을 대상으로 실 LLM 슬라이스(claude/codex 라우팅 + `harnessGuardCheckPII`/Assertions 재검)를 구축해 놓았다 — **김건우 히어로는 재시드가 필요 없다**(오태림 F3와의 핵심 차이). PR#1을 머지만 하면 최단 경로로 성립한다.
- **로컬모델 인프라**: EXAONE·Qwen 등 로컬모델 서빙 환경이 데모 전 준비돼야 §3의 PII 하이브리드가 완전히 성립한다. 미준비 시 "로컬 불가 → codex 폴백"으로 정직하게 강등해 시연한다.
- **감사 해시체인 이식 여부**: §4에서 정직 표기한 대로, 해시체인 검증 UI가 JB_project2에 없다면 F1 데모의 감사 탭은 "로그 존재"까지만 보여줄 수 있고 "무결성 검증" 시연은 예선 MVP로 대체해야 한다.
