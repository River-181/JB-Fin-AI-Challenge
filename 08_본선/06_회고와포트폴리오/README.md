# 06 회고와 포트폴리오

> 회고(원료)와 포트폴리오(가공물)는 별개 활동이 아니다. 회고 없이 쓴 포트폴리오는 자랑만 남고, 포트폴리오로 정제하지 않은 회고는 다음 프로젝트에 아무것도 넘겨주지 못한다. 이 폴더는 그 둘을 하나의 파이프라인으로 연결한다.

## 목적

JB Fin:AI Challenge(팀 망상궤도/JByond) 참가 경험을 정직하게 회고하고, 그 회고에서 건진 사실·수치·의사결정을 **이력서·면접·포트폴리오에 바로 쓸 수 있는 형태**로 남긴다.

## 흐름

```
01_회고        각자, 그리고 팀이 무슨 일이 있었는지 정직하게 기록 (원료)
   ↓
02_종합        회고에서 반복되는 패턴·거시/미시 인사이트를 추출 (가공)
   ↓
03_포트폴리오   종합된 인사이트 중 "대외적으로 보여줄 수 있는 것"만 추려 정리 (결과물)
```

## 용도

- **개인**: 이직·면접 준비할 때 `03_포트폴리오/{이름}.md` 하나만 열면 됨. 근거(수치·의사결정·커밋)는 전부 `04_증빙/`에 실측 기록이 있으므로 면접에서 "숫자 어디서 나왔냐"는 질문에 실제로 답할 수 있다.
- **팀**: `01_회고/팀-회고.md`에 팀 전체 KPT를 남기고, 각자 `01_회고/개인-회고/{이름}.md`에 자기 관점(팀 전체 의견과 다를 수 있음)을 남긴다.

## 원칙

- **회고와 포트폴리오는 신뢰 수준이 다르다.** 회고는 주관·감정·불만도 그대로 남긴다(다만 "이건 내 개인 의견"이라고 표시). 포트폴리오는 증빙 가능한 사실·수치만 쓴다.
- **개인 회고의 주관은 팀 전체 입장이 아니다.** `01_회고/개인-회고/`의 글은 작성자 1인의 시각이며, 팀 전체 평가로 인용하지 않는다.
- **숫자는 `04_증빙/`을 인용한다.** 새 숫자를 여기서 만들지 않는다 — [[08_본선/04_증빙/02_분석자료/팀-기여-통계-20260705|팀 기여 통계]] 등 기존 실측치를 링크로 참조.
- **팀원 누구나 자기 폴더를 채울 수 있다.** `03_포트폴리오/_템플릿.md`을 복사해 `{이름}.md`로 저장하면 된다.
- **4번째 팀원 이름 미확정.** 볼트 문서마다 이재형/재형/최영욱으로 다르게 적혀 있고 사용자가 구두로 언급한 "김재형"도 어디에도 정확히 일치하지 않는다 — `03_포트폴리오/_TBD-4번째팀원.md` 참고, 확정 전까지 추측해서 파일명에 쓰지 않는다.
