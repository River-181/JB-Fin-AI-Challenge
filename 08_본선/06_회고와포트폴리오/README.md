# 06 회고와 포트폴리오 — 관제탑

> **이 폴더 하나가 프로젝트 전체의 입구다.** 팀원은 프로젝트 폴더를 다 뒤질 필요 없이 여기서 모든 산출물·문서·데모·통계로 진입하고, 종합판 정보를 본다. 그리고 **이 폴더만 떼어 공유해도** 사람이든 AI(Claude)든 이걸 읽고 취업·발표에 쓸 자료(지원서·인터뷰 답변·스토리)를 뽑아낼 수 있다 — 핵심 사실·수치·서사를 폴더 안에 내장했기 때문이다.

## 여기서 무엇을 할 수 있나

- **처음 왔다면** → [[08_본선/06_회고와포트폴리오/00_개요/프로젝트-랩업|프로젝트 랩업]] 한 장으로 "우리가 뭘 했고 뭘 남겼는지" 파악.
- **경험을 글·답변으로 꺼내 써야 하면** → `03_포트폴리오/{내 이름}.md` + [[08_본선/06_회고와포트폴리오/03_포트폴리오/스토리뱅크|스토리뱅크]].
- **AI에게 이 폴더를 통째로 주고** "이 자료로 협업 경험 에세이 써줘 / 발표용 STAR 뽑아줘"라고 하면, [[08_본선/06_회고와포트폴리오/00_개요/팩트시트|팩트시트]]의 내장 수치·서사만으로 답이 나온다(폴더 밖 파일 불필요).

## 흐름

```
01_회고        각자·팀이 무슨 일이 있었는지 정직하게 기록 (원료)
   ↓
02_종합        회고에서 거시/미시 인사이트를 추출 (가공)
   ↓
03_포트폴리오   증빙 가능한 것만 추려 지원·발표용으로 정리 (결과물)
```

## 원칙

- **회고와 포트폴리오는 신뢰 수준이 다르다.** 회고엔 주관·불만도 남기되 "개인 의견" 표시. 포트폴리오는 증빙 가능한 사실·수치만.
- **개인 회고의 주관은 팀 전체 입장이 아니다.** `01_회고/개인-회고/`는 작성자 1인의 시각.
- **SSOT는 `04_증빙/`. 단 이 폴더는 자립해야 하므로** 핵심 수치는 `00_개요/팩트시트`에 출처를 달아 내장한다.
- **팀원 누구나 자기 폴더를 채운다.** `03_포트폴리오/_템플릿.md`을 복사해 `{이름}.md`로.
- **4번째 팀원 이름은 이재형으로 확정(2026-07-07).**
