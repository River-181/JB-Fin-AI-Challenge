# 미시 회고

> 목적: 기술·프로세스 단위의 구체적 교훈만 담는다. 팀·커리어 관점은 [[08_본선/06_회고와포트폴리오/02_종합/거시-회고|거시-회고]] 참고.

## 미시적 인사이트 (기술·프로세스 관점)

### 1. 벌크 치환 사고 — 위키링크 경계와 유니코드 정규화(NFC/NFD)

7/5, 다른 세션이 리서치 폴더 재구성 후 README 링크를 고치려다가 위키링크 `[[ ]]` 밖의 일반 텍스트 속 "README" 단어까지 문자열 그대로 치환해버렸다. `AGENTS.md`·`decision-log.md`·`session-log.md`, 심지어 append-only 과거 기록인 `프롬프트-로그.md`까지 **38개 파일 113곳**이 오염됐다(`08_본선/04_증빙/01_핵심로그/session-log.md:217`).

복구는 두 단계였다. 1차로 셸 문자열 치환(sed)을 돌렸는데 27개 파일은 성공했지만 11개 파일에서 실패했다. 원인을 추적하니 한글 자모가 결합형(NFC)과 분해형(NFD)으로 섞여 있어 바이트 단위 문자열 매치가 어긋난 것이었다. 2차로 Node.js `String.normalize('NFC')`를 파일 내용과 검색 문자열 양쪽에 적용한 뒤 재치환해 나머지 11개 파일 29곳을 마저 복구했고, 전체 볼트를 정규화 포함 재스캔해 잔존 오염 0건을 확인했다(`session-log.md:218`).

→ 교훈: 한글 텍스트를 대상으로 벌크 치환 스크립트를 짤 때는 (1) 위키링크 `[[ ]]` 경계를 문자열 매치 범위에서 반드시 제외하고, (2) 치환 전후로 `normalize('NFC')`를 적용해야 한다. macOS 파일시스템이나 일부 동기화 도구가 파일을 NFD로 저장하는 경우가 실제로 있었고, 그게 이 사고의 직접 원인이었다.

### 2. 병렬 에이전트 운영 — 자원 충돌과 "되돌리지 않고 기다렸다 이어받기"

7/4, canon-moc-sync 잔여 경고 4종을 병렬 Haiku 서브에이전트 4개로 위임하던 중 도달성 미달 지표가 일시적으로 2건→118건으로 폭증했다(`session-log.md:204`). 조사해보니 다른 동시 세션이 같은 리서치 폴더를 플랫 구조에서 하위폴더 구조(`01_프롬프트`·`02_결과-원문`·`03_요약-정리`)로 재구성하는 중이었다 — 내 에이전트 작업과는 완전히 무관한, 의도된 리팩터였다.

이때 택한 원칙은 "다른 세션의 재구성 자체는 되돌리지 않는다"였다. 재확인 시점에는 그 세션도 자체적으로 죽은 링크·도달성 문제를 완결한 상태였고, 남은 미처리 파일 1건만 확인해 수동으로 마저 고쳤다.

→ 교훈: 같은 저장소를 여러 AI 세션이 동시에 건드릴 때, 충돌을 감지하면 롤백보다 "관찰 후 기다렸다 이어받기"가 더 안전할 수 있다. 다만 이 프로토콜은 사전에 파일·폴더 소유권을 나누지 않은 상태에서 즉흥적으로 작동한 것이라, 다음번엔 벌크 편집 전에 "이 경로를 건드리는 세션이 있는지" 확인하는 절차를 표준화해야 한다 — [[08_본선/06_회고와포트폴리오/01_회고/팀-회고|팀 회고]] Next Time #2가 이 지점을 이미 액션으로 잡아뒀다.

### 3. 리스크 레지스터 — 정적 문서만으로는 부족했다

[[08_본선/02_전략/03_decisions/risk-register|리스크 레지스터]]는 6/26에 R1(시연 기술 장애)~R5(경쟁팀 유사 아이디어) 5개 리스크를 사전 정의했다. 그런데 "상세 대응 방안"과 "신규 리스크 추가" 섹션은 본선 종료 시점까지 "작성 예정"으로 비어 있었다(`risk-register.md:36,40,46`).

실제로는 레지스터에 없던 리스크(위 #1의 README 오염 사고)가 현장에서 새로 발생했는데, 이걸 레지스터에 기록하고 확산을 막는 루틴 자체가 없었다 — 사고 대응은 session-log에는 잘 남았지만, "다음 리스크 점검 때 이걸 반영한다"는 피드백 루프로는 이어지지 않았다.

→ 교훈: 사전에 잘 만든 문서라도 "매 세션 시작 시 1분 점검" 같은 살아있는 루틴과 묶이지 않으면 사실상 죽은 문서가 된다. 예측은 했지만, 현장에서 그 예측을 검증하는 습관까지는 붙지 않았다.
