## 예선 MVP와 본선 제품 코드는 다른 실행 경로다 — 헷갈리지 말 것

| 구분 | 위치 | 실행 |
|---|---|---|
| 예선 MVP | 이 모노레포 `02_제품/app/` (vanilla JS 정적 앱) | `npm install && npm run dev` → `http://127.0.0.1:8000/index.html?demo=sme`(전주 중앙로 카페 히어로) · `?demo=jeonse` · `?demo=phishing` |
| 본선 제품 코드 | 별도 레포([[08_본선/06_회고와포트폴리오/00_링크허브\|링크 허브]] "제품 코드 레포 정본") — 이 모노레포에는 코드가 없다 | [[08_본선/06_회고와포트폴리오/00_링크허브\|제품 라이브 데모]](RM 업무보드 실동작), 로컬 실행은 해당 레포 README 참고 |

## 순서

1. [[08_본선/03_제품/docs/02_core-bet|Core Bet]] — 왜 이 문제인가 (30초). "챗봇이 아니라 운영 콘솔을 만든 이유"부터 연다.
2. [[08_본선/03_제품/05_diagrams/99_comprehensive-architecture|종합 아키텍처]] — Case→AgentRun→Evidence→Approval→Audit 구조가 어떻게 동작하는가.
3. 제품 라이브 데모([[08_본선/06_회고와포트폴리오/00_링크허브|링크 허브]] "제품 라이브 데모" 링크, RM 업무지원 포털 `#/roles/rm-officer/board`) — 실제로 켜서 케이스 하나를 승인까지 넘겨본다. 예선 히어로 시나리오를 함께 보여줄 땐 `?demo=sme`로 전주 중앙로 카페 케이스를 연다.
4. [[08_본선/03_제품/reports/구현현황-JB_project2|구현현황 리포트]] — 실제/목업 경계를 먼저 자백한다. "메인 판단 루프는 결정론적 mock, 로컬모델(Ollama) 경로는 하네스의 별도 '샘플 요청' 버튼을 눌러야 실행되는 opt-in", "RM 콘솔 1곳만 해시체인 감사가 실제로 동작"처럼 구체적으로 말한다.
5. [[08_본선/06_회고와포트폴리오/01_회고/팀-회고|팀 회고]] — 아쉬운 점까지 준비돼 있음을 보여준다.
