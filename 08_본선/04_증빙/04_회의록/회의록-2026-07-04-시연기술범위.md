## 안건 = 결정 게이트

### G2. 구동 형태 — 배포형 / 로컬형 / 하이브리드
- 백엔드 유무·로컬모델 시연 방식·인프라가 여기 종속.
- 💡 실무 제안: **하이브리드** — 로컬 모델 실동작(은행 보안 체감)은 온디바이스, 그 외 흐름은 정적/브라우저로 시연 완주 우선.

### G3. DB 범위 — 정적 / 서버API+영속DB / 하이브리드
- Case→AgentRun→Evidence→Approval→Audit 영속·감사체인 구현 깊이.
- 💡 실무 제안: **최소 하이브리드** — 핵심 경로만 영속(감사체인 시연), 나머지 localStorage. 본선 시간 제약.

### G6. 연동 범위 — 실 API / 목업 / 혼합
- P1 데이터 기반·"실동작" 신뢰도. 계열사=G1 종속.
- 💡 실무 제안: **혼합** — 1개 도메인은 실 공개 API(신뢰 임팩트), 나머지 목업.

### 골든패스 시연 시나리오 (G4 In 도메인 기반)
- 도메인별 대본: 신호수집 → 판단 → 행동초안 → **담당자 승인** → 감사기록. 로컬모델 실동작 포인트 표시.
- 💡 제안: 골든패스 2~3종 + 각 30~60초 스크립트, 실패 시 폴백 화면.

### 시연 역할·폴백
- 발표/조작/해설 분담, 네트워크·모델 실패 시 폴백(녹화/정적 스냅샷). 조직도 메인 UI 진입 데모.

### 리허설 일정 (7/4~5 현장)
- e2e 완주 리허설 슬롯, 최종 덱 확정, 심사 Q&A 예상.
