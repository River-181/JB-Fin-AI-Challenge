## 딥리서치 프롬프트 (이 아래 통째로 붙여넣기)

당신은 **AI 에이전트의 메모리·지식표현 아키텍처를 정리하는 학술 리서처**다. 목표는 "한 *케이스(사안·사건 단위)*의 지식·근거·판단 이력을 잘 **기록·데이터화·표현·재활용**하는 에이전트를 어떻게 설계하는가"에 대한 **선행연구·설계 패턴(일반론)**을 동료평가 논문·학술 자료에 근거해 정리하는 것이다. 결과는 "케이스 단위 메모리를 가진 거버넌드 AI 에이전트 콘솔"의 설계를 학술적으로 정당화하는 데 쓰인다.

**중대 검색 규칙(엄수)**: **"paperclip", "Hermes/헤르메스"라는 단어로 검색하지 말 것.** 이 단어들은 동음이의(사무용품·여러 회사·신화·금융상품·여러 LLM)가 너무 많아 엉뚱한 결과를 진짜처럼 인용하게 만든다. 오직 **일반 학술 명칭**으로만 검색하라: episodic memory(일화기억), semantic memory, case-based reasoning(CBR, 사례기반추론), agent memory, long-term memory for LLM agents, memory-augmented agents, RAG over case history, knowledge representation for cases, provenance/audit trail in AI. **정의가 명확하지 않은 고유명사·제품명은 인용하지 말고 [미검증]으로 처리**하라.

### 알아내야 할 것 (핵심 질문)

**A. 에이전트 메모리의 유형론 (선행연구 일반론)**
1. AI 에이전트/LLM 에이전트의 **메모리 유형 분류**: 단기/작업기억(short-term/working) vs 장기기억(long-term), **일화기억(episodic)·의미기억(semantic)·절차기억(procedural)** 구분이 에이전트 설계에 어떻게 적용되나. 각 유형의 정의·역할·대표 연구.
2. **장기기억을 가진 LLM 에이전트의 대표 아키텍처·서베이**: 메모리 쓰기(저장)·읽기(검색)·갱신·망각(forgetting)·요약(consolidation)의 라이프사이클. 대표 서베이/프레임워크가 제시하는 메모리 모듈 구조.

**B. 케이스 기반 추론(CBR)과 케이스 표현**
3. **사례기반추론(Case-Based Reasoning)의 고전 프레임워크**: Retrieve–Reuse–Revise–Retain(4R) 사이클, "케이스"를 어떻게 표현(문제–해결–결과 구조)하고 색인·검색·재사용·축적하는가. CBR이 새 사안을 과거 케이스로 푸는 원리.
4. **CBR과 LLM/RAG의 현대적 결합**: 과거 케이스 이력 위의 RAG, LLM 에이전트에 CBR을 결합한 최근 연구. "한 케이스의 지식을 데이터화해 다음 케이스에 재활용"하는 패턴.

**C. 케이스 지식의 표현·데이터화 (핵심)**
5. **한 케이스 단위의 지식을 구조화·데이터화하는 표현 방식**: 지식그래프/온톨로지, 구조화 스키마(문제·근거·판단·결과·다음행동), 출처·근거 연결(provenance), 시계열 이벤트 로그. 어떤 표현이 검색성·설명가능성·재사용성에 유리한가.
6. **근거·추적성(provenance/audit trail)을 메모리에 내장**하는 연구: 에이전트의 각 판단에 근거·출처·불확실성·"다음 확인 항목"을 함께 기록하는 설계 패턴, 설명가능성(explainability)과의 연결.

**D. 메모리 활용 — 검색·재사용·학습**
7. **메모리를 판단에 활용하는 메커니즘**: 관련 과거 케이스/메모리 검색(retrieval), 컨텍스트 주입, 메모리 기반 계획·반성(reflection)·자기개선(self-improvement) 루프(예: 경험을 요약해 향후 행동을 개선하는 에이전트). 대표 연구의 작동 원리.
8. **메모리의 위험·한계**: 오염된 메모리·잘못된 케이스 재사용·확증편향·프라이버시(케이스에 민감정보 누적)·메모리 비대화. 메모리 아키텍처의 알려진 실패모드와 완화책. (최소 1건 한계·실패 의무 수집.)

**E. 금융·고위험 도메인의 케이스 메모리**
9. **금융·법률·의료 등 고위험·규제 도메인에서 케이스 메모리/사례 지식을 다룬 연구**(있다면): 케이스 단위 기록·재사용이 감사·규제·설명가능성과 어떻게 연결되는가. 직접 연구가 희소하면 일반 도메인으로 보강하되 "직접 사례 부재"를 명시.
10. *(범용 덤 유도)* 위 외에, "협업·생산성 도구의 메모리"(여러 에이전트·사람이 한 케이스 지식을 공유·누적하는 구조)나, 케이스 메모리를 잘 설계해 차별화한 **일반적 설계 원칙**이 있으면 추가 카테고리화하라(고유 제품명 없이 원리만).
