# 담당자 유형별 대시보드·스킬·에이전트 근거기반 설계 (기획 프롬프트)

> ⚠️ 대외비. 승인 대기(proposed) 문서 — 아래 신규 Agent/Skill은 전부 `product`의 "제안→승인" 게이트를 거치기 전까지 `skill-registry.md` / `01-에이전트-프롬프트-계약.md`에 반영하지 않는다.
> 근거: 딥리서치 27종(`D1a`~`Dplus-b`, `_결과/`) 전량 + 2026-07-01 실시간 웹서치(각 담당자 유형별 국내 실사례) + `work/02_제품/app/app.js`(현재 구현된 6개 역할 대시보드) 코드 확인.
> 목적: 이미 설계된 하네스(Case→Agent→Skill→Evidence→Approval→Audit, PII 비반출, L0–L4 승인레벨 도출규칙, 기존 14 에이전트/25 스킬)는 그대로 두고, UI가 앞서가 스킬 레지스트리와 어긋난 4개 대시보드(기업여신·소비자보호·FDS·준법감시)의 갭을 리서치 근거로 메운다.

## 0. 고정 하네스 — 변경 금지 (그대로 유지, 이 문서는 이 위에 "추가"만 한다)

- **파이프라인**: Case → Agent → Skill → Evidence → Approval → Audit. 고객 영향 행동은 승인 게이트 통과 전 자동 실행 금지.
- **공통 출력 스키마(전 에이전트 강제)**: `judgment`(판단) / `evidence`(근거) / `confidence`(불확실성) / `nextChecks`(다음 확인) / `approvalPolicy`(승인레벨).
- **승인레벨 L0–L4는 "장착 스킬의 Approval Policy"에서 도출**(직접 지정 금지): internal only→L0, RM/advisor review·approval required→L1/L2, mandatory·legal review→L3, blocks external action→L4. 에이전트 레벨 = 장착 스킬 중 최고 레벨.
- **기존 14 에이전트 / 25 스킬**(`work/03_에이전트/01-에이전트-프롬프트-계약.md`, `skill-registry.md`)은 삭제·재정의하지 않는다. 아래 제안은 전부 신규 "추가"이며, 기존 경계 규칙(§2)과 PII 비반출·가명처리 원칙을 그대로 상속한다.
- **거버넌스**: 08_본선 `AGENTS.md`상 신규 Agent/Skill 정의·범위 변경은 `product`의 "제안→승인" 게이트를 거쳐야 한다. 이 문서의 모든 신규 항목은 **승인 대기(proposed)** 상태로 표기하고 임의 반영하지 않는다.
- **수치 표기 원칙**: `_canon.md` SSOT 규칙을 그대로 따른다. 모든 신규 수치는 출처 칩(1차/2차/[미검증]/[추정])을 붙인다. 절대값 KPI("100%", "0건")는 "관측된 증거" 언어로 표현한다(§8-2 근거).

## 2. RM 담당자

### 2-1. Pain Point (내부 근거)
- RM은 프런트(상담)·미들(심사보조)·백오피스(사후관리)를 동시 수행하며 3대 병목은 ① 비정형 서류(재무제표·세금계산서) 수기 대조, ② 여신심사보고서 정성 의견 매번 신규 작문, ③ EWS·기한이익상실(EOD) D-day를 계좌 수백 건 중 추적 실패 [D2].
- 이자연체 14일 경과 시 EOD 발동, D-3영업일 전 서면통지 의무 — 누락 시 채권보전 흠결 [D2]. 사업자대출 사후점검 기준이 2026.6.30부로 1억→5천만원(개인)으로 하향, 점검 케이스가 급증 [D2].
- Baseline: RM 시간당 완전부담원가 [추정] 5.5만~12.9만원, 만기연장 1건 30분 이상(2차, 단일출처) [D16].
- **한국 금융 RM 직무에 대한 AI 생산성 직접 RCT는 전무** — 이 공백 자체가 핵심 리스크다 [D8]. BCG/METR 연구는 "AI 프런티어 밖" 낯선 복합 과업에서 오히려 -19%p 정답률/속도 저하를 보고한다 — 오버셀 금지 [D8].

### 2-2. 실제 업계 해법 → 매핑
| 업계 사례 (출처) | 우리 제품 매핑 |
|---|---|
| KB국민은행 AI 에이전트 100여개(PB·RM·여신심사), 신한은행 고객관리·자산관리·여신심사 에이전트 순차 투입 ([우리은행 기업금융·WM AI 고도화](https://v.daum.net/v/20260127142641253)) | 기존 **RM Copilot Agent**(L1/L2) + **Cashflow Triage Agent** 조합이 이미 이 패턴과 정합 — 신규 에이전트 불필요, 하단 KPI를 baseline 수치로 교체 |
| 우리은행 5대 영역(상담·자동화·WM/RM·기업여신 E2E·내부통제) 재설계 | RM/기업여신/준법감시 대시보드가 이미 이 5대 영역 분해와 구조적으로 일치함 — 설계 방향 확인됨 |

### 2-3. 지시
- KPI 카드의 예시값("7건/2건/6건/4건" 등)을 D16 baseline("RM 시간당 원가 [추정]", "만기연장 30분+[2차]")로 교체하고 출처 칩을 붙인다.
- 신규 에이전트 불필요. 기존 RM Copilot / Cashflow Triage / Policy Match 유지.

## 3. 기업여신 담당자 (신규 Agent 제안 다수)

### 3-1. Pain Point
- 심사 흐름: 상담(운전자금 vs 시설자금) → 신용분석(CB+CSS) → 임점실사 → 품의서 → 전결/본부 handoff(약 60%가 지점전결, 2차) → 약정·기표 → 사후관리 [D2].
- 담보: LTV×감정가 - 선순위채권 - 방공제 = 유효담보가액. 지방은행 SME 여신은 신보/기보/지신보 보증서가 핵심 신용보강 [D2].
- 자금용도외 유용 점검이 대출 후 3개월 내 의무, 2026.6.30부 개인사업자 5천만원 초과 전수점검, 1차 적발 시 1~3년/2차 적발 시 5~10년 신규대출 전면 제한(1차 규정) [D2].
- JB 규모: FY2025 총자산 73.1조원, 전북은행 대출금 19.08조원/연체율 1.46%/NPL 1.12%(다수 1차, 일부 [미검증]) [D3a].

### 3-2. 실제 업계 해법 → 매핑
| 업계 사례 (출처) | Pain point | 제품 기능 매핑 |
|---|---|---|
| 하나은행: 생성형AI 기업신용평가 심사의견 30분→10초, 전 영업점 도입 ([인사이트코리아](https://www.insightkorea.co.kr/news/articleView.html?idxno=242939), [이코노미트리뷴](https://www.economytribune.co.kr/news/articleView.html?idxno=3902023)) | 심사의견 정성 작문 반복 | **신규 Credit Document Analyst Agent** — 재무제표·정성의견 초안 |
| 신한은행: 머신러닝 기반 '기업여신 자동심사 지원시스템(Bics)' | 신용분석 반복 | 상환위험 분류(기존 Cashflow Triage) + 신규 담보가치 스킬 |
| OCR+RPA로 서류검수·데이터입력 자동화, 평균 심사기간 2~3일 단축 ([koreadeep.com](https://www.koreadeep.com/blog/ai-loan-auto)) | 서류대조 병목 | 신규 `document-ocr-crosscheck` 스킬 |

### 3-3. 신설 Agent/Skill 제안 (승인 대기)

**여신서류·재무분석 에이전트 / `Credit Document Analyst Agent` · L1/L2 (제안)**
- 장착 스킬(신규): `financial-statement-parse`(재무제표·부가세·세금계산서 파싱, RM review), `collateral-valuation`(LTV·유효담보가액 산정, RM review), `facility-lease-review`(시설자금/리스·할부 CREFIA 공시 대조, RM review)
- system: 재무·담보·공시 자료를 대조해 심사 패킷을 만든다. 신용 결정을 자동화하지 않는다.
- task: 입력=재무제표(범위화)·담보자료 → 산출=심사 의견 초안 + 담보가치 산정 근거.

**전결 라우팅 에이전트 / `Delegation Routing Agent` · L1/L2, 고위험시 L3 (제안)**
- 장착 스킬(신규): `delegation-authority-router`(전결한도 매트릭스는 파라미터DB로 처리 — 은행별 비공개이므로 상수화 금지 [D4 갭])
- system: 전결 가능/본부심사/여신위원회 후보를 규칙 기반으로 분기한다. 임계값은 하드코딩하지 않고 설정값으로 관리한다.

**자금용도 사후점검 에이전트 / `Fund Usage Audit Agent` · L1/L2, 위반의심시 L3 (제안)**
- 장착 스킬(신규): `fund-usage-check`(세금계산서·송금내역 대조), 기존 `document-checklist` 재사용
- system: 자금용도외 유용 신호를 탐지한다. 제재(1~3년/5~10년 제한) 확정은 사람이 결정한다.

### 3-4. 지시
- 위 3개 에이전트를 `01-에이전트-프롬프트-계약.md`에 "제안" 섹션으로 추가(승인 전 반영 금지).
- 대시보드 "통제 원칙" 패널에 "전결 매트릭스는 파라미터, 상수 아님" 문구 추가 — D4 갭 반영.

### 4-2. 지시 (신규 Agent 불필요 — 이미 충분)

- 대시보드 "외부 참고 기준" 패널에 국토부 안심전세앱·서울시 AI분석보고서를 레퍼런스로 명기(경쟁이 아니라 공적 데이터 정합성 근거로 인용).

## 5. 소비자 보호 담당자 (신규 Agent 제안)

### 5-1. Pain Point + 업계 해법
- 금융소비자보호법: 적합성 원칙(재산·투자경험 기반 적합상품만 권유) + 적정성 원칙(고객 자발적 구매도 부적정시 경고), 손실감수능력 등 **6개 필수확인정보 "종합" 고려** 의무 ([금융위 보도자료](https://fsc.go.kr/no010101/84056)).
- 기업은행 'AI 불완전판매 탐지 시스템'(투자경험·거래이상패턴 종합분석), 우리은행+KT 'AI기반 불완전판매 방지 프로세스'(실시간 판매과정 분석), 신한은행 AI 완전판매 프로세스(실시간 진단·직원 알림) ([디지털타임스](https://www.dt.co.kr/article/12065369), [파이낸셜뉴스](https://www.fnnews.com/news/202112261749525864)).
- 한국법상 자동화 판단에 대한 설명요구·이의제기권이 법제화(신용정보법§36의2, 개인정보법§37의2) → "완전자동 승인/거절"이 아니라 **인간승인+사후감사**가 법적으로도 정합적 [D14].

### 5-2. 신설 Agent/Skill 제안 (승인 대기)

**적합성·적정성 심사 에이전트 / `Suitability Screening Agent` · L1/L2, 부적합시 L3 (제안)**
- 장착 스킬(신규): `suitability-check`(6개 필수확인정보 대조), `product-risk-match`
- system: 상품 위험과 고객 상황 불일치 후보를 표시한다. 권유 적합/부적합을 "확정"하지 않는다.

**부당권유 차단 에이전트 / `Misselling Guard Agent` · L3 mandatory (제안, 기존 `compliance-guard` 스킬 확장 사용)**
- 장착 스킬(신규): `prohibited-claim-detector`(확정수익·손실축소·과장 표현), 기존 `claim-limiter` 재사용
- system: 고객 발송 전 문구를 검토한다. 통과 전 외부 발송 차단.

**민원 대응 에이전트 / `Complaint Response Agent` · L1/L2 (제안)**
- 장착 스킬(신규): `complaint-classifier`, `response-draft`(근거패킷+답변초안)

**취약고객 보호 에이전트 / `Vulnerable Customer Agent` · L1/L2 (제안)**
- 장착 스킬(신규): `vulnerable-customer-flag`, `plain-language-rewrite`

### 5-3. 지시
- 4개 에이전트를 "제안" 상태로 계약 문서에 추가. `compliance-guard`/`audit-ledger`는 기존 것 재사용(신규 승인 스킬 아님).

## 6. 보이스피싱/FDS 담당자 (신규 Agent 제안 — 기존 Fraud Shield 단일 에이전트로는 세분화 부족)

### 6-1. Pain Point (수치, D1b)
- 2025년 1~11월 발생 21,588건 · 피해액 1조 1,330억원(전년동기 대비 건수 +15.6%, 피해액 +56.1%) [D1b, 금융위 2025-12-30].
- **범행유형 구조전환**: 기관사칭형 57.0% vs 대출사기형 43.0% — "대출사기→기관사칭"으로 급격 이동 [D1b].
- 정책효과: 2025.8.28 종합대책 이후 2025.10~2026.4 7개월 연속 전년동기대비 발생·피해액 각 **35.3% 감소**. ASAP AI 정보공유 플랫폼이 26.6만건 공유로 약 419억원 선제 차단 [D1b].

### 6-2. 실제 업계 해법 → 매핑
| 업계 사례 (출처) | 매핑 |
|---|---|
| 신한금융 그룹통합FDS: 혁신금융서비스 지정, 은행·카드·증권·라이프 간 실시간 이상거래 정보 공유, 2주간 의심정보 1,111건 분석·41건 탐지·8억원 피해 예방 ([1코노미뉴스](https://www.1conomynews.co.kr/news/articleView.html?idxno=48433)) | 준법감시 대시보드의 "계열사 데이터 공동이용" 요건(D17 클린룸 모델)과 결합 필요 — FDS 신호공유는 §7 계열사 경계 설계를 그대로 따라야 함 |
| 카카오뱅크 FDS AI 고도화 ([뉴스핌](https://www.newspim.com/news/view/20260123000574)), 인뱅3사+금융보안원 연합학습모델(2026.6 적용, 개별 탐지→"전 금융권 공동학습·공동방어" 전환) | 대시보드의 "공동 탐지룰" 배지가 이미 이 방향과 일치 — 연합학습 방식임을 명시 |
| Wells Fargo 결함 자동필터로 100만개+ 계좌 오차단(CFPB 제재) [D15] | 오탐 해제 SLA를 반드시 넣어야 하는 반례 근거 |
| 국내 오탐 사례: 금감원이 소명자료 충분시 **5영업일 내** 심사결과 통보로 표준화 [D15] | Freeze Review Agent의 SLA 값으로 채택 |

### 6-3. 신설 Agent/Skill 제안 (승인 대기)

**이상징후 패턴탐지 에이전트 / `Anomaly Signal Agent` · L0/L1 (제안)**
- 장착 스킬(신규): `impersonation-pattern`(기관사칭·원격제어앱·딥페이크), `mule-account-pattern`(대포통장·분산출금)
- system: 신호를 수집·스코어링만 한다. 차단 실행 권한은 없다 — 실행은 기존 Fraud Shield Agent(L4)가 보유.

**지급정지·해제 심사 에이전트 / `Freeze Review Agent` · L3 mandatory (제안)**
- 장착 스킬(신규): `freeze-justification-check`(오탐 소명 심사, **5영업일 SLA**[D15 근거]), `interbank-liaison`(타행 공조 기록)
- system: 해제는 반드시 사람 승인. 해제 오판의 배상 리스크(Wells Fargo 사례)를 감사 로그에 남긴다.

**탐지룰 운영 에이전트 / `Detection Rule Ops Agent` · L0/L1 (제안)**
- 장착 스킬(신규): `rule-performance-monitor`(공동룰/자체룰 성능, override 비율), `rule-update-proposal`

### 6-4. 지시
- 이상거래 KPI에 "채택/승인 건수"가 아니라 **override 비율·사유, false-block 해제 SLA, kill-switch 발동시간**을 핵심 지표로 채택(D15 KPI 권고 그대로).
- Fraud Shield Agent는 라인 리드(L4, 차단 전결)로 유지, 위 3개 신규 에이전트가 하위에서 신호/해제/운영을 분담.

## 7. 내부통제 준법감시 담당자 (신규 Agent 제안)

### 7-1. Pain Point + 법령 근거
- 3선 방어 체계(1선 RM/영업점, 2선 리스크·준법·소비자보호, 3선 감사)가 한국 은행 조직의 뼈대 [D4, 법률신문].
- **책무구조도**: 2024~2026 전격 도입, 전북은행도 2024.10.30 제출·시범운영 개시 — 임원별 책무 고유 지정으로 통제 균열 봉쇄 ([뉴스핌](https://www.newspim.com/news/view/20241030001063)) [D4].
- 신용정보법 §40조의2 / PIPA §28조의5: 가명정보-추가정보 분리보관, **재식별 시 즉시 회수·처리중지 의무** [D5a].
- 금융지주회사법 §48조의2: 계열사간 정보공유는 "내부 경영관리 목적"에 한정되며 **상품소개·구매권유 목적은 명시적으로 제외** — FDS 신호공유·RM 영업활동에 그룹데이터를 쓰면 특례 이탈 위험 [D17].
- 감사증거: append-only, 해시체인, 시각동기화, chain-of-custody 필요(대법원 판례상 원본-출력물 동일성 요구) [D15]. 접속기록 보존 신용정보업감독규정 1년+백업, 전자금융감독규정 5년 [D5a/D15].
- 금융위 'AI 위험관리 프레임워크(AI RMF)' 2026년 1분기 확정 예정 — 책무구조도에 AI 관련 내부통제·리스크관리 책무를 반영하도록 요구(2026-07-01 웹서치 검증).

### 7-2. 신설 Agent/Skill 제안 (승인 대기)

**책무구조도 매핑 에이전트 / `Accountability Mapping Agent` · L1/L2, 위반시 L3 (제안)**
- 장착 스킬(신규): `duty-map-trace`(업무-담당자-승인자-책임임원 연결), `three-lines-check`(1/2/3선 이행 확인)

**PII/가명처리·계열사경계 통제 에이전트 / `Data Boundary Guard Agent` · L3 mandatory (제안)**
- 장착 스킬(신규): `pii-egress-scan`, `pseudonymization-audit`(분리보관·접근권한 점검), `affiliate-boundary-check`(금융지주회사법§48-2 목적제한 검증 — 영업/마케팅 액션에 그룹데이터 사용 여부 감시)

**감사원장 무결성 검증 에이전트 / `Audit Integrity Agent` · L0/L1, 위반발견시 L3 (제안)**
- 장착 스킬(신규): `hash-chain-verify`, `approval-log-completeness`

**사고대응·킬스위치 에이전트 / `Incident Response Agent` · L4 (제안)**
- 장착 스킬(신규): `kill-switch-trigger`, `incident-report-draft`(전자금융감독규정시행세칙 3단계 보고: 최초 24시간 이내·중간·종결)

### 7-3. 지시
- 4개 에이전트 "제안" 등록. `compliance-guard`/`audit-ledger`는 기존 것 재사용.
- P0/S0 장애등급(PII 반출의심·감사원장 쓰기실패)은 즉시 Fail-Closed + 15분 내 통보로 대시보드 "통제 원칙" 패널에 명시(D19 근거).

## 8. 전 페르소나 공통 설계 원칙 (대시보드 공통 개선 지시)

### 8-1. 신뢰보정 UX (D18 근거 — 6개 대시보드 공통 적용)
- confidence score만으로는 trust calibration 불충분(Liao, FAT* 2020). 모든 work-card에 **"왜 지금 떴는지 근거 링크 + 불확실성 + 대안행동"**을 evidence-first로 노출.
- 한 번의 명백한 오류로 신뢰가 급락(Dietvorst 2014) → AI 산출물은 "그대로 승인" 버튼이 아니라 **수정 가능한 제안**으로 설계(이미 승인 게이트 구조와 정합, UI 문구만 보강).
- override 사유 분류체계(EVID_MISSING/FALSE_POSITIVE/TRAINING_GAP 등)를 6개 대시보드 공통 컴포넌트로 도입.

### 8-2. KPI 표준화 (D13/D18 근거)
- 채택률 단독 KPI 금지. 공통 KPI 세트: `approval_lead_time`, `rework_rate`, `override_reason_distribution`, `alert_ignore_rate`, `follow_up_completion`.
- "PII 비반출 100%", "근거추적성 100%" 같은 절대값 표현은 "사전 정의된 critical flow 내 0 failures observed"처럼 **관측된 증거 언어**로 재작성(HHS/NIST 권고, D13).

### 8-3. 구조 근거 (D11 — 이미 구현된 하네스의 정당화 자료, 변경 아님)
- Case→AgentRun→Skill→Evidence→Approval→Audit는 각각 CBR의 New Case/Episodic Memory/Procedural Memory/Semantic Memory/Revise/Retain에 대응 — 발표 시 "왜 이 구조인가"의 학술적 근거로 인용 가능.

### 8-4. TCO/SLA (D19 — 준법감시 대시보드 반영)
- 12개월 상업 밴드[후보, 미검증]: 파일럿 1.5~2.8억, 정식운영 3.0~5.0억, 강화운영 5.0~8.0억+. SLA는 파일럿 99.0~99.5%, 정식운영 99.9%.

### 8-5. 경쟁 차별점 재확인 (D+a)
- 신한 '여신심사지원 Agent'(단일도메인 자체구축), MS Copilot/Salesforce FSC Copilot(범용, 통제기능 無), 세이프홈즈(전세 단일), 토스 FDS(단일) 대비 **"PII 비반출×승인우선 HITL×멀티도메인(소상공인·전세·FDS·소비자보호·준법)" 결합 상용 제품은 부재 — White Space**로 확인됨. 발표에서 그대로 사용 가능.
