# D1a — 소상공인 금융부담·연체·폐업 (사실 수치)

## 메타 (프롬프트 밖)
- **도메인**: 소상공인/자영업자 금융부담·연체·폐업의 **사실 수치**(규모·심각도·추세) + RM 여신 업무 baseline 단가/빈도.
- **티어**: ★★★
- **심사항목**: 1 (주제적합성·문제정의) — 특히 1.3 Pain Point / 1.4 문제 중요도.
- **실행 배치**: 배치 1 (사실 선행, 병렬). 후속 프롬프트가 이 수치를 인용.
- **경계 메모**:
  - **D1(여기)=수치(얼마나 아픈가)**, **D2=업무 메커니즘(어떻게 일하는가)**. 여신·심사 절차의 *작동 방식* 설명은 금지(D2가 담당). 단, **"건당 손실·소요시간·1인당 처리건수" 같은 정량 baseline은 여기**(수치이므로).
  - 전세사기·보이스피싱 피해 수치는 **D1b**가 담당. 여기서 다루지 않음.
  - JB금융그룹 자기 수치(연체율·포트폴리오)는 **D3a**. 여기는 시장 전체·산업 통계.

---

## 딥리서치 프롬프트 (이 아래 통째로 붙여넣기)

당신은 한국 금융·자영업 통계를 1차 출처까지 추적하는 **금융 리서치 애널리스트**입니다. 목표는 한국 소상공인·자영업자의 **금융부담·연체·폐업** 실태를 *수치로* 입증하고, 동시에 은행 RM(여신 담당)의 **여신 업무 baseline(건당 손실·소요시간·1인당 처리건수)**을 가능한 한 정량으로 확보하는 것입니다. 이 수치들은 "RM용 거버넌드 AI Agent 콘솔"의 문제정의 근거로 쓰입니다 — 따라서 *왜 이 문제가 비싸고 중요한가*를 숫자로 뒷받침해야 합니다.

### 검증 앵커 (먼저 이 수치들을 검증·최신화하시오)
아래는 내가 이미 보유한 baseline입니다. **각 항목을 1차 출처로 검증하고, 더 최신 수치가 있으면 갱신하고, 출처를 1차화(원 발행기관 원문)** 하시오. 틀렸으면 어디가 어떻게 틀렸는지 명시.
1. 자영업자 대출 잔액 **1,064.2조원** (한국은행, 2024말 추정)
2. 취약 자영업자 **42.7만명 = 전체의 13.7%** (한국은행, 2024말)
3. 자영업자 연체율 — 전체 **1.67%** / 비은행 **3.43%** / 취약차주 **11.16%**
4. 은행 개인사업자(자영업자) 연체율 **약 0.72–0.78%** (금융감독원, 2025)
5. 2024년 폐업 **100.8만명** (첫 100만 돌파), 폐업률 **9.04%**, 폐업 사유 중 '사업부진' **50.2%** (국세청)

### 구체 질문 목록 (가이드 — 단, 가이드 밖이라도 의미 있는 수치를 발견하면 반드시 추가 보고)
**A. 규모·잔액**
- 자영업자/개인사업자 대출 잔액 총량과 최근 분기 추이(은행 vs 비은행 구분). 다중채무 자영업자 비중·규모.
- 취약 자영업자(저신용·다중채무·저소득 교집합) 정의와 규모·비중. 정의 기관별 차이.

**B. 연체·부실 심각도**
- 자영업자 연체율: 전체/은행/비은행/취약 구분의 최신 분기 수치와 1년·2년 추세.
- 은행권 개인사업자 대출 연체율·고정이하여신비율(NPL)·부실채권 규모.
- 연체 → 부실 → 대위변제/상각으로 가는 단계별 전이율(roll rate)이 공개돼 있다면 수집.

**C. 폐업·생존**
- 연간 폐업자 수·폐업률 추이(최근 3~5년), 업종별·연령별 분포, 폐업 사유 분포(특히 '사업부진'·'금융부담').
- 창업 대비 폐업 비율, 신생기업 1·3·5년 생존율.

**D. RM 여신 업무 baseline (정량 단가·빈도 — 최우선 확보 대상)**
- **여신 1건당 평균 연체/부실 손실액**(LGD·대위변제 평균액 등 어떤 형태로든 단가화 가능한 수치).
- **여신/신용 심사 1건 소요시간** 또는 사후관리 1건 소요시간(은행 실무·연구·감독자료·언론 추정 무엇이든).
- **RM(여신담당) 1인당 관리 여신 건수/차주 수/포트폴리오 규모** — 시중은행·지방은행 평균.
- 사후관리(모니터링) 누락·지연으로 인한 손실 사례·규모, 조기경보(EWS) 미작동 통계.
- ※ 이 D항목은 공개 통계가 희소할 수 있음 — **추정·간접지표·언론 보도라도 출처와 함께** 보고하되 신뢰도를 [2차]/[미검증]으로 정직하게 표기.

**E. 추세·반전 신호 (덤 발견 유도)**
- 금리·매출(카드매출 등)·물가가 자영업 부실에 미치는 영향 관련 최근 분석.
- 위 어디에도 안 들어가지만 "이 문제가 크고 비싸다"를 입증하는 **놀라운 수치**가 있으면 별도 섹션으로.

### 출처 규칙 (반드시 준수)
**공통 5가드**
1. 발견마다 **출처 URL + 원문 짧은 인용 + 발행처·발행일 + 신뢰도(1차/2차/[미검증])** 를 붙인다.
2. 핵심 수치는 **2개 이상 출처로 교차검증**. 단일 출처면 [미검증] 표기.
3. **링크에 실제 접속 못 하면 그 수치는 인용 금지**(상상·추정 출처 금지).
4. **한국 맥락·최신 기간 우선**. 오래된 수치는 연도를 명시하고 최신본을 함께 찾는다.
5. 서로 충돌하는 수치는 둘 다 제시하고 발행처·산정방식 차이를 설명.

**도메인 가드 (1차 출처 우선순위)**: 한국은행(금융안정보고서·ECOS), 금융감독원, 금융위원회, 국세청·통계청, 소상공인시장진흥공단, 신용보증기금/지역신용보증재단을 **1차**로 본다. 연구소·신문·벤더 리포트는 **2차**. 1차 원문에 도달하면 2차 인용을 대체할 것.

### 출력 형식 (이 구조로)
1. **핵심 발견 요약** (5~8개 bullet, 각 한 줄 + 핵심 수치)
2. **근거표** — 열: `주장 | 수치 | 출처 URL | 발행처·날짜 | 신뢰도(1차/2차/미검증) | 원문 인용(한 줄)`
3. **앵커 검증 결과** — 위 5개 앵커 각각에 대해 `확인됨 / 갱신됨(신값+출처) / 반박됨(근거)`
4. **제품 적용 시사점** — 각 수치가 "RM용 AI Agent 콘솔"의 문제정의를 어떻게 강화하는지 (특히 D항목 baseline → ROI 논거).
5. **갭·[미검증] 목록** — 못 찾은 수치, 출처가 약한 수치, 1차화 실패 항목을 정직하게 나열.

### 깊이·범위 지시
Gemini/Perplexity Deep Research로 실행. 표면 요약 금지 — 1차 통계 원문(보고서 PDF·보도자료·통계표)까지 파고들 것. 특히 **D항목(RM baseline)은 공개 데이터가 흩어져 있으므로** 연구보고서·감독원 검사자료·은행 IR·학술 논문·언론 심층기사를 폭넓게 훑어 단가/빈도를 한 개라도 더 확보하는 데 집중. 최소 12개 이상 독립 출처 목표.

## Opus 점검 로그 (2패스)

### 패스 1 — 적대적 비판 → 보완
**비판**
- (a) D항목(RM baseline)이 "수치 vs 메커니즘" 경계를 넘을 위험. "여신 심사 소요시간"은 자칫 D2(업무 메커니즘)와 겹친다 — 엔진이 절차 설명으로 새면 D2와 중복 산출.
- (b) 앵커만 검증시키면 엔진이 "앵커 재확인"에 매몰돼 *새로운* 수치(덤)를 안 가져올 수 있음. 발견 인센티브 부족.
- (c) "취약 자영업자 13.7%"의 분모가 불명확(전체 자영업자 차주 수? 대출자 수?) — 엔진이 분모를 멋대로 가정할 위험.
- (d) 출처 우선순위에 소상공인시장진흥공단·신보를 안 넣으면 폐업·보증 통계 1차원을 놓침.

**보완**
- (a) 경계 메모와 D항목 설명에 **"작동 방식 설명 금지, 정량 baseline만"**을 명시. D는 "단가·빈도·건수" 같은 *숫자*로만 한정.
- (b) E항목 + 각 질문 블록 머리말에 "가이드 밖이라도 의미 있는 수치 발견 시 추가 보고", "놀라운 수치는 별도 섹션" 지시 삽입 → 덤 발견 유도.
- (c) A항목에 "정의 기관별 차이", 근거표에 원문 인용 열을 둬서 분모·정의를 강제로 드러내게 함.
- (d) 도메인 가드 1차 목록에 소진공·신보·지역신보 추가.

### 패스 2 — 재비판 → 보완
**재비판**
- (e) D항목이 "희소할 수 있다"고만 하면 엔진이 쉽게 포기하고 빈손 보고 → 가장 중요한(ROI 논거) 데이터가 비는 최악.
- (f) 출력 형식에 "앵커 검증 결과"가 근거표와 분리돼 있어, 엔진이 앵커를 근거표에서 빠뜨릴 수 있음(이중 누락).
- (g) 신뢰도 표기를 강제하지만 [2차]/[미검증]을 *정직하게* 쓰라는 압력이 약함 — 엔진이 모두 1차로 부풀릴 유인.

**보완**
- (e) D항목에 "추정·간접지표·언론이라도 출처와 함께 보고하되 신뢰도 정직 표기"를 추가하고, 깊이 지시에서 "D항목에 집중·최소 출처 수 목표"를 못박음 → 포기 방지 + 정직성 양립.
- (f) "앵커 검증 결과"를 출력 형식 3번 독립 섹션으로 유지하되 *근거표(2번)에도 포함*되도록 앵커가 곧 주장임을 질문 구조상 자연스럽게 연결(앵커 항목들이 A~C 질문과 1:1 대응).
- (g) 공통 가드 4·5 + D항목·갭 섹션에서 "정직하게 [미검증] 표기", "1차화 실패 항목 나열"을 반복 강조해 부풀리기 비용을 높임.
- **수렴**: 경계(수치 한정)·발견 인센티브·정직한 신뢰도·1차 출처 우선이 모두 프롬프트에 내장됨. 최종 확정.
