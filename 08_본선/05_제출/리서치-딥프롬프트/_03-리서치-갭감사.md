# 리서치 갭 감사 — JB LocalGuard OS

> ⚠️ 대외비. 로컬 산출물 기반 감사. 웹검색은 사용하지 않고, 지정 파일·종합 검증본·표본 결과만 확인했다.

## 총평

23종 분해는 심사 5범주를 넓게 덮고 있다. 특히 PII 비반출(D5a·D5b·D10), 승인우선(D15 예정·D11·D13), 멀티도메인(D1a·D1b·D7a), JB 적합성(D3a~d·D3b·D3c)은 방향이 맞다.

다만 "1위"를 넘어 **JB가 수억을 주고 살 수준**으로 보이려면, 현재 리서치의 약점은 기술 아이디어보다 **구매 의사결정 숫자와 도입 운영성**이다. D+a의 ROI는 구조는 있으나 행동가정이 강하고, D2는 RM 업무흐름은 자세하지만 시간·원가 baseline이 부족하다. D3a는 SOHO/개인사업자 여신 잔액, 계열사별 DART 직접표, 지역·상품별 포트폴리오 수치가 비어 있다.

따라서 추가 리서치는 이미 존재하는 D14·D15·D3d, 흡수 후보(D+a 조달/KPI 민감도, D5b AI 기본법), 백로그(밸류에이션 배수·AI운영보험·lock-in·규제보고·경쟁 M&A)를 재제안하지 않는다. 새 후보는 "도입하면 얼마가 줄고, 누가 쓰며, 어떤 데이터 경계와 SLA로 운영되는가"에 집중한다.

### D16 프롬프트 골자
- 한국 은행 RM/기업여신/개인사업자 여신 업무의 공개 가능한 시간·인력·처리량 baseline을 수집한다.
- 품의서 작성, 증빙 대조, EWS 대응, 자금용도외 유용 점검, 만기/연체 관리별 평균 소요시간과 변동폭을 찾는다.
- 공개 수치가 없으면 인접 업무 연구·노무자료·은행 채용/직무자료를 근거로 추정 라벨을 붙인다.

### D17 프롬프트 골자
- 신용정보법·PIPA·금융지주/계열사 데이터 처리 관점에서 전북은행·광주은행·JB우리캐피탈 간 데이터 이동 가능/불가능 경계를 분해한다.
- PoC에서 허용 가능한 데이터셋, 가명/익명 처리, 처리위탁, 공동이용 동의, 클린룸/토큰화 운영 모델을 표로 만든다.
- "외부 LLM 비반출"뿐 아니라 "그룹 내부 계열사 간 과잉 공유 금지"를 방어한다.

### D18 프롬프트 골자
- RM이 AI 제안을 언제 신뢰/거부하는지, 자동화 편향·알림 피로·책임소재 회피가 금융 의사결정에 미치는 영향을 조사한다.
- 승인우선 UI에서 필요한 friction, explanation, uncertainty, override, feedback, training 설계를 도출한다.
- KPI는 채택률 단독이 아니라 승인까지 리드타임, 재작업률, override 사유, 경보 무시율로 둔다.

### D19 프롬프트 골자
- 외부 LLM 호출비, 프록시/DLP/로그 저장, 감사원장 보관, 보안성 평가, 운영 인력, 장애대응을 포함한 TCO를 산정한다.
- 금융권 B2B PoC→파일럿→운영 전환 시 SLA, RTO/RPO, 보안 패치, 모델 버전 변경 통보, 장애 커뮤니케이션 기준을 정리한다.
- 수억 원 구매안을 "라이선스"가 아니라 "운영 가능한 서비스 계약"으로 보이게 만든다.

## 논문 필요 영역

1. **케이스 메모리 아키텍처**: CBR, episodic/semantic memory, provenance KG, memory poisoning, long-term agent memory를 금융 케이스 단위로 연결하는 근거 보강.
2. **생산성 측정**: RCT, crossover, staggered rollout, DID, speed-quality-accuracy-rework 동시 측정. D8의 일반 문헌을 RM 파일럿 설계로 번역해야 함.
3. **공정성·신용편향**: 여신/모기지/신용평가 LLM 편향, fairness metric, adverse action explanation, protected class proxy 문제. D14 회수 후에도 학술 인용 보강 필요.
4. **Human-AI trust calibration**: 자동화 편향, 알림 피로, override behavior, expert-in-the-loop decision support. 승인우선 콘솔 UX의 핵심 근거.
5. **RAG·근거충실성 평가**: faithfulness, groundedness, citation precision/recall, claim support ratio, judge-model bias. D13 지표와 D10 보안을 연결.
6. **보안·프롬프트 인젝션 실증**: indirect prompt injection, tool-call injection, DLP false positive/negative, red-team benchmark. "완전 차단"이 아니라 residual risk 관리로 표현.

## 권고 액션

1. **즉시**: D3d·D14·D15 회수부터 완료한다. 현재 하 등급 3개가 남아 있어 23종 완결성이 깨진다.
2. **즉시**: D3a의 SOHO/개인사업자 여신 잔액, FY2025 총자산 73.1조, 계열사 포트폴리오 수치를 DART/IR PDF로 1차화한다.
3. **즉시**: D+a 슬라이드용 숫자는 "단일 추정치/보수 시나리오/행동가정" 라벨을 붙이고 Alloyx 오연결 수치는 제거 또는 재확인한다.
4. **다음 리서치**: D16과 D17을 우선 실행한다. D16은 가격 방어, D17은 멀티도메인·그룹 확장 방어에 직접 연결된다.
5. **문서 반영**: 제출서에는 "PII 비반출"만 말하지 말고 "계열사 간 데이터 경계", "승인 전 행동 차단", "감사 가능한 근거 체인", "RM 업무량 baseline 대비 절감"을 한 세트로 제시한다.
6. **표현 정제**: D+b 계열 문장 중 "100% 탕감", "전격 발효"처럼 검증 부담이 큰 표현은 "위험 경감", "도입/논의/시행 여부 확인 필요"로 낮춘다.
