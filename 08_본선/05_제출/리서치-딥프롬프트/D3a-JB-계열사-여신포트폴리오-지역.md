# D3a — JB · 계열사 · 여신포트폴리오 · 지역

## 메타 (프롬프트 밖)
- **도메인**: D3a — JB금융그룹 계열사·사업·고객접점·지역기반·여신 포트폴리오 **사실**.
- **티어**: ★★★
- **심사 항목**: 2 (금융업무·사업 연계성 및 고객가치) — 100점의 단독 근거 클러스터.
- **실행 배치**: 배치 1 (사실 선행, 병렬 — 후속 프롬프트가 인용할 KPI·수치 확보).
- **경계 메모**:
  - D+a와 경계: **JB 자기 사실 = D3(여기) / 경쟁사·타행·핀테크 = D+a**. 본 프롬프트는 JB 내부 사실만. 시장점유율은 "JB의 지역 점유율" 한정(경쟁 비교표는 D+a).
  - D3b와 경계: **계열사·여신·지역 = D3a(여기) / AX·디지털 전환·KPI·뉴스 = D3b**.
  - SSOT: `_canon.md` §5(계열사)·§10(JB 사업 연계 사실)이 충돌 시 승리.

## 딥리서치 프롬프트 (이 아래 통째로 붙여넣기)

당신은 JB금융그룹을 분석하는 **금융 IR·여신 포트폴리오 리서치 애널리스트**다. 산출물의 청중에 JB금융그룹 관계자(심사위원)가 포함된다고 가정하라. 따라서 위키 수준 개요는 0점이며, **DART 전자공시·IR 자료 수준의 공시 KPI 깊이**(잔액·비중·점유율·연체율·순익·ROE 등 구체 수치)를 요구한다.

### 역할·목표
JB금융그룹과 5개 계열사(전북은행·광주은행·JB우리캐피탈·JB자산운용·JB인베스트먼트)의 **계열사 구조·사업영역·고객접점·지역기반·여신 포트폴리오**를 공시 수준 수치로 확정하라. 목표는 "JB가 어떤 고객(특히 지역 소상공인·개인사업자·전세 고객)을, 어떤 여신 규모로, 어느 지역에서 응대하는가"를 RM(영업점 관계관리자) 업무 맥락이 보일 만큼 정량화하는 것이다.

### 확정해야 할 앵커 사실 (이미 알려진 baseline — 1차 출처로 재확인·정밀화하라)
- 계열사: **전북은행 · 광주은행 · JB우리캐피탈 · JB자산운용 · JB인베스트먼트** (이 5개. 그 외를 계열로 추가할 때는 지분율·근거 명시).
- **FY2025 그룹 순이익 약 7,104억원(사상 최대), ROE 약 12.4%(7년 연속 두 자릿수)** → DART/IR로 정확 수치·기준(지배주주 순이익 여부)·발표일 확인.
- **전북은행 소상공인 대환대출 취급 1위(2024년 687억원)** → 출처·집계기준 확인.
- 광주은행 광주·전남 상생금융.

### 구체 질문 (가이드 — 더불어 인접 고가치 사실도 자유롭게 발견하라)
1. **그룹 재무 골격**: FY2025(및 가능하면 직전 분기/연도 추세) 그룹 순이익·ROE·ROA·BIS비율·NIM. 그룹 총자산(보도상 약 66~67조원으로 거론되나 **1차 미확정 → DART 사업보고서/연결재무제표로 정확 수치를 재확인**). 각 수치의 기준일·발표 주체 명기.
2. **계열사별 규모·역할**: 전북은행·광주은행·JB우리캐피탈 각각의 총자산·순이익·자기자본, 그룹 내 손익 기여 비중. JB자산운용·JB인베스트먼트의 사업영역(운용 AUM·투자 분야)과 그룹 내 위치.
3. **여신 포트폴리오(핵심)**: 전북은행·광주은행의 **소상공인/개인사업자(SOHO) 여신 잔액·전체 여신 대비 비중**, 가계·기업 여신 구성. 가능하면 중소기업·가계신용 세분. JB우리캐피탈 포트폴리오 구성(자동차금융·기업금융·개인신용 등 비중).
4. **지역 점유율·기반**: 전북은행의 전북 지역, 광주은행의 광주·전남 지역 **여수신/대출 점유율 또는 지역 내 위상**. 지역 점포 수·고객 기반. (경쟁 타행과의 전국 비교는 제외 — JB의 지역 내 위상에 한정.)
5. **자산건전성 추이**: 그룹·은행별 **연체율·고정이하여신(NPL)비율 최근 추이**(가능하면 분기별), 특히 소상공인/개인사업자 부문의 연체율 동향과 충당금. 추세의 방향(악화/개선)과 배경.
6. **고객접점·상생금융 프로그램**: 전북은행·광주은행이 운영하는 소상공인·자영업자·전세 관련 금융상품·상생금융·정책금융 연계(대환대출·이자환급·보증연계 등)의 실제 명칭과 규모.
7. (덤) RM 업무 흐름에 관여하는 사실: 영업점/RM 조직 규모, 기업여신 상담→심사→사후관리 프로세스 관련 공시·보도.

### 출처 규칙 (반드시 준수)
- **1차 출처 우선순위**: 재무·여신·점유율·연체율 등 수치는 **DART 전자공시(dart.fss.or.kr) 사업보고서·분기보고서·연결재무제표, JB금융지주 IR 자료(실적발표 PT·보도자료)** 만 1차로 인정. 보도(언론) 수치는 **[2차]** 로 표기하고 1차로 교차 확인되지 않으면 별도 표기.
- 발견마다: URL + 원문 짧은 인용 + 발행처·날짜 + 신뢰도(**1차 / 2차 / [미검증]**).
- 핵심 수치(순익·ROE·여신잔액·점유율·연체율)는 **2개 출처 교차**. 단일 출처면 [미검증] 표기.
- 링크에 실제 접속 못 하면 그 수치는 인용하지 말 것.
- **한국 맥락·최신 우선**: 가장 최근 공시(FY2025 / 최신 분기)를 우선. 오래된 수치는 연도 병기.
- **제약(반드시 준수 — _canon SSOT)**:
  - **네이버클라우드 MOU(2025-12-26)는 "추진 방향·검토 단계"로만 표현하라. "이미 운영 중"·"도입 완료"라고 단정 금지.** (사후관리 등 기업여신 전반 AI 적용은 단계적 검토 단계.)
  - **핀크(Finnq)를 JB금융 계열사로 표기 금지(미확인).** 계열사는 위 5개로 한정.
  - **그룹 총자산 수치는 보도값(약 66~67조)을 그대로 쓰지 말고 DART에서 재확인할 것.** 재확인 안 되면 [미검증].
- _canon.md와 충돌 시 _canon이 승리(SSOT).

### 출력 형식
1. **발견 요약**: 도메인별(재무·계열사·여신·지역·건전성·고객접점) 핵심 수치를 불릿으로.
2. **근거표**: `| 주장(수치) | 출처 URL | 발행처·날짜 | 신뢰도(1차/2차/[미검증]) | 원문 인용(짧게) |`
3. **제품 적용 시사점(JB KPI 연결)**: 각 발견이 RM용 거버넌드 AI Agent 콘솔(소상공인·전세 고객의 위험신호→Case→AI 판단→사람 승인) 설계에 주는 함의를 1~2줄씩. "전북은행 SOHO 여신 X억·연체율 Y% → 상환위험 분류·정책금융 매칭의 잠재 대상 규모" 식으로 JB 사업 규모와 연결.
4. **갭 / [미검증] 목록**: 1차 출처로 확인 못 한 항목, 재확인 필요 수치(특히 총자산), 추가 리서치가 필요한 빈칸.

### 딥리서치 깊이 지시
- DART 정기공시(사업보고서·분기/반기보고서)와 JB금융지주 IR 실적자료를 **직접 열어** 표 단위 수치를 인용하라. 요약기사 재인용으로 끝내지 말 것.
- 가능한 한 분기 시계열로 연체율·여신잔액 추이를 제시하고, 소상공인/개인사업자 부문을 별도로 분리하라.
- 전북·광주은행의 지역 기반(전북 / 광주·전남)을 정량(점유율·점포·고객)으로 드러내라.

## Opus 점검 로그 (2패스)

**패스 1 (적대적 비판 → 보완)**
- 비판 A — "위키 깊이 위험": 초안이 '계열사 나열 + 순익 1줄'에 머물면 심사 2번에서 위키 수준=0점. → 구체 질문을 여신 잔액·비중·연체율 시계열·지역 점유율로 강제하고, 출력형식에 "분기 시계열·SOHO 분리"를 명시.
- 비판 B — "환각 자석(총자산)": 보도값 66~67조를 그대로 인용하면 DART 충돌 위험. → 제약에 "총자산 DART 재확인, 안 되면 [미검증]" 박음. 핀크 오기재·MOU 과장 두 제약도 인라인.
- 비판 C — "D+a 침범": 점유율 질문이 타행 비교로 새면 SSOT 충돌. → 4번 질문을 "JB의 지역 내 위상"으로 한정하고 경계 메모에 명시.

**패스 2 (재비판 → 보완)**
- 재비판 D — "제품 연결 누락": 수치만 모으면 심사위원이 'so what'을 못 봄. → 출력형식 3번에 'JB KPI/사업 규모 → 제품(상환위험 분류·정책금융 매칭) 잠재 대상'으로 연결하는 시사점 섹션을 의무화.
- 재비판 E — "1차/2차 혼입 위험": 보도 수치가 1차로 둔갑할 소지. → 출처 규칙에 'DART/IR만 1차, 보도는 [2차] 명시 + 2출처 교차'를 못박고 근거표 신뢰도 컬럼 강제.
- 재비판 F — "RM 맥락 빈약": 여신 수치만으로는 RM 콘솔 정당화가 약함. → 6·7번 질문에 상생금융 프로그램·RM 조직/프로세스(사후관리) 덤 발견을 추가해 업무 접점 정량화 유도.
- 수렴: 수치 깊이 강제 + 환각 가드(총자산·핀크·MOU) + 제품/KPI 연결 + 경계(D+a) 정합 확보.
