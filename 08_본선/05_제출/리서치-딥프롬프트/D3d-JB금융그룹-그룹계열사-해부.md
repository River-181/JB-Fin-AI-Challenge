# D3d · JB금융그룹 그룹·계열사 해부 (낱낱이)

> ⚠️ 대외비 — 6/29 공식발표 전 비공개.

**도메인** JB금융그룹 (D3 토대 레이어) · **티어** ★★★ · **심사** 2(연계성) · **배치** 1 · **엔진** 🔵 Gemini
**경계**: D3a=여신·소상공인 problem-fit 수치 / D3b=AX·디지털 전략 / D3c=도입 정합·트리거 / **D3d=그룹 전체 해부**(정체성·지배구조·각 계열사 사업 특성·실적·고객·강약점). 여신 problem 수치(→D3a)·AX 전략(→D3b)은 여기서 깊이 파지 말 것 — *그룹·계열사 구조 자체*가 대상.

## 딥리서치 프롬프트 (이 아래 통째로 붙여넣기)

당신은 **금융지주 산업 분석가**입니다. 목표는 **JB금융그룹이 어떤 그룹인지, 그리고 각 계열사의 사업 특성을 낱낱이 해부**하는 것입니다. 결과는 "RM용 거버넌드 AI Agent 콘솔"을 JB에 제안하는 팀이 *고객(JB)을 깊이 안다*는 것을 증명하는 토대 자료로 쓰입니다. 위키 수준 요약이 아니라, **각 계열사의 사업모델·고객군·상품·실적·강약점**까지 구체적으로.

### 핵심 질문
**A. 그룹 정체성·지배구조**
- JB금융지주 설립 연혁(지주 전환), 본사 소재(전북 전주), 그룹 비전·경영철학·슬로건.
- **지배구조**: 최대주주·주요 주주 구성(지분율), 외국인 지분, 이사회·경영진(회장·주요 CEO), 행동주의 펀드 등 거버넌스 이슈.
- 그룹 전체 규모: 총자산·자기자본·시가총액·임직원 수·점포 수. (총자산 정확수치는 **DART 재확인 전 [미검증]**)

**B. 계열사별 해부 (각 사마다 ▸사업영역 ▸주 고객군 ▸주력 상품/서비스 ▸채널·점포 ▸최근 실적(자산·순익·건전성) ▸시장 지위 ▸강점·약점)**
- **전북은행** (지방은행, 전북 기반)
- **광주은행** (지방은행, 광주·전남 기반)
- **JB우리캐피탈** (여전사 — 자동차금융 등)
- **JB자산운용**
- **JB인베스트먼트** (VC/PE)
- **디지털·핀테크 관련 자회사/조직** (있는 것만; ⛔ **핀크(Finnq)는 JB 계열로 표기 금지 — 미확인**)
- **해외 법인** (예: 전북은행 캄보디아 PPCBank, 광주은행 해외 등 — 실재·지분·사업 확인)

**C. 사업 포트폴리오·수익 구조**
- 그룹 수익의 은행 vs 비은행(캐피탈·운용·VC) 비중, 이자/비이자 비중.
- 그룹 차원의 건전성(NPL·연체)·자본비율(BIS/CET1) 개요.

**D. 지역 기반·정체성**
- 전북·광주/전남 지역에서의 위상·점유율·지역경제 연계, "지역 상생/관계금융" 정체성.

**E. 경쟁 포지셔닝**
- 타 지방은행지주(BNK·DGB/iM) 대비 그룹 규모·수익성·특성 비교(간단 표).

**F. 덤(범용 여지)**: 위에 없지만 그룹을 이해하는 데 중요한 특이점(M&A 이력, 효율성 지표 ROE/CIR 강점, 특화 사업, 최근 조직개편 등)을 별도 보고.

### 출처 규칙 (반드시 준수)
1. 발견마다 **출처 URL + 원문 짧은 인용 + 발행처·날짜 + 신뢰도(1차/2차/[미검증])**.
2. **재무·지분·실적은 DART 전자공시·각 사 IR·사업보고서·공식 홈페이지를 1차**로. 언론은 [2차], 핵심 수치는 2출처 교차.
3. 연도를 반드시 명시(최신 회계연도 우선). 추정은 [추정] 분리.
4. ⛔ **핀크(Finnq) JB 계열 표기 금지**. 그룹 총자산 정확수치는 DART 확인 전 [미검증].
5. 링크 실접속 못 하면 인용 금지. 한국 맥락·최신.

### 출력 형식
1. **그룹 한눈 요약** (정체성·규모·지배구조 5~7 bullet)
2. **계열사 해부 표** — 행=계열사, 열: `사업영역 | 주 고객군 | 주력 상품 | 채널·규모 | 최근 실적(자산·순익·건전성) | 강점 | 약점`
3. **수익구조·건전성 개요**
4. **지역 기반·경쟁 포지셔닝**
5. **근거표** — `주장 | 출처 URL | 발행처·날짜 | 신뢰도 | 원문 인용`
6. **갭·[미검증]** — 미확인·약한 출처 정직 표기

## Opus 점검 로그 (2패스)
- **패스1 (중복·범위)**: D3a(여신 수치)·D3b(AX 전략)와 겹칠 위험 → 경계메모+역할문에 "그룹·계열사 *구조 자체*가 대상, 여신 problem 수치·AX 전략은 양보"로 고정. "낱낱이"를 충족하도록 계열사별 7속성(사업·고객·상품·채널·실적·강·약) 표를 강제.
- **패스2 (환각·출처)**: 핀크 오귀속·총자산 환각·계열사 누락이 위험 → 핀크 금지·총자산 [미검증]·해외법인(PPCBank 등) 실재확인 지시 인라인. 지분·실적 환각 차단 위해 DART·IR 1차 + 2출처 교차 + 연도명시 강제. 경쟁 비교(E)는 "간단 표"로 한정해 D+a(경쟁·시장)와 중복 방지.
