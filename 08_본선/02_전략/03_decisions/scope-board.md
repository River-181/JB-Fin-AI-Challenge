# Scope Board

> 본선 MVP 범위 결정. In/Out을 명확히 해서 범위 크리프(scope creep)를 방지한다.

---

## In Scope (본선 시연 포함)

- [ ] 위험 신호 → Case 자동 생성 (Triage Agent)
- [ ] Case 대시보드 (5컬럼 칸반: New/진행/검토/완료/차단)
- [ ] 에이전트 스킬 실행 + 라이브 스트리밍
- [ ] 인간 승인 게이트 (Approval 인터페이스)
- [ ] 에이전트 실행 로그 (AuditEvent)
- [ ] 전세사기 위험 탐지 시나리오 (핵심 시연)

---

## Out of Scope (본선 이후)

- 실제 고객 데이터 연동 (규제·보안 이슈)
- 다채널 발송 (문자·이메일) 실제 연동
- 모바일 앱 인터페이스
- 멀티 지점 동시 지원
