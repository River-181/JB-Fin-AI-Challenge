# 01 · 케이스 상세 페이지 (자율운영 뷰)

> 모듈: `views/caseDetailPage()` · 진입: 케이스 클릭 → 우측 패널이 아닌 **전용 뷰**로 진입

## 문제
현재는 무엇을 눌러도 우측 상세 패널 요약만 뜬다. 실제로는 케이스를 "들어가서" 각종 기능 버튼과 에이전트 작동, 산출물, 추적을 보는 전용 페이지가 필요하다 (HagentOS 상세 페이지식).

## 진입/이탈
- 케이스 카드/행 클릭 → `activeView="case-detail"`, `selectedCaseId` 설정 → `caseDetailPage()` 렌더
- 상단 breadcrumb: `케이스 보드 / JBG-104 전주 중앙로 카페` (뒤로가기)
- 우측 패널은 보조(거버넌스/감사 요약)로 유지하되, 본문이 주가 된다

## 레이아웃 (섹션 = 모듈)
1. **헤더 바**: 코드·고객명·계열사·리스크 점수·상태 배지 + 액션 버튼들
   - `에이전트 실행` `승인 요청` `승인` `반려` `산출물 생성` `고객 추적 보기` `JSON 내보내기`
   - 버튼은 케이스 상태에 따라 활성/비활성 (기존 핸들러 재사용)
2. **자율운영 타임라인**: 감지→근거수집→분석→초안→승인게이트→감사 단계별 진행. 각 단계에 담당 에이전트·사용 스킬·소요·상태.
3. **에이전트 루프 패널**: 배정 에이전트별 현재 작업/큐/예산. 실행 버튼.
4. **근거(Evidence) + 플러그인 소스**: 이 케이스가 참조한 법령·정책·뉴스·JB DB 항목 (플러그인 레지스트리 연동).
5. **산출물(Deliverables)**: 생성된 MD 결과물 목록 + 뷰어(03 스펙). "일을 시켰으면 결과물" 가시화.
6. **데이터 거버넌스 적용**: 이 케이스 처리에서의 토큰화/모델 라우팅/외부 반출 결과 (07 스펙).
7. **고객 추적 요약**: 이 케이스의 고객, 연결된 과거/현재 케이스 타임라인 (06 스펙).
8. **승인 게이트 + 감사 원장**: 기존 gates/audit 재사용, 스크롤 가능.

## 데이터 인터페이스
`caseDetailPage(caseObj)` — 케이스 객체 + 레지스트리(plugin/skill/deliverable/customer/governance) 조회만으로 렌더. 케이스 스키마 외 의존 없음 → 다른 케이스에도 그대로 적용.

## 갈아끼우기
섹션은 독립 렌더 함수(`renderCaseTimeline`, `renderCaseDeliverables` 등)로 분리. 순서/표시는 배열 설정으로 제어.

## 심사 효과
③ 에이전트가 실제로 일하는 루프, ④ 결과물·검증가능성을 한 화면에서 증명.
