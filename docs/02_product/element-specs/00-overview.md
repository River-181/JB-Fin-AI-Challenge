# 업그레이드 개요 — 모듈 레지스트리 아키텍처

> 작성일 2026-06-14 · JB LocalGuard OS · Fin:AI Challenge 제출 직전 업그레이드
> 원칙: **모든 요소를 레지스트리에 등록 → 나중에 갈아끼우기(swappable). 모든 데이터·외부연결은 거버넌스 계층을 통과.**

## 1. 왜 바꾸는가 (심사축 매핑)

| 심사축 | 현재 한계 | 업그레이드 |
| --- | --- | --- |
| ③ 에이전트 설계·기술 타당성 | 우측 패널 요약만, 케이스가 "처리"되는 모습 약함 | 히어로 케이스 자율운영 상세 뷰 + 에이전트 루프 |
| ④ MVP 완성도·검증가능성 | 일을 시켜도 결과물이 안 보임 | 케이스별 산출물 MD(실제 결과물) + 보드 처리 훅 |
| ② 금융·사업 연계 | 스킬이 한 줄 라벨뿐, 외부 데이터 없음 | 실질 스킬 콘텐츠 + 플러그인(법령·정책·뉴스·JB DB) |
| ⑤ 혁신·확장·리스크 관리 | PII/외부모델 리스크 미설계 | **데이터 거버넌스 계층**(차별점) + 모듈 레지스트리 확장성 |

## 2. 모듈 경계 (각 요소 = 1 스펙 파일)

```
app.js
├─ registries/                  ← 갈아끼우는 지점
│   ├─ pluginRegistry[]         05-plugin-mcp-registry.md
│   ├─ skillRegistry[]          02-skill-content-engine.md
│   ├─ agentRegistry[]          (기존 agents 유지)
│   └─ deliverableRegistry{}    03-case-deliverables.md
├─ governance/                  07-data-governance-pii.md  ← 모든 외부호출 관문
├─ customers[]                  06-customer-db-tracking.md
└─ views/
    └─ caseDetailPage()         01-case-detail-view.md
보드 처리 훅                      04-board-hook.md
UX/UI 수리·스크롤                 09-ux-ui-fixes.md
README·다이어그램                 08-readme-diagrams.md
```

각 모듈은 **(a) 데이터 인터페이스 (b) 렌더 함수 (c) 거버넌스 계약**만 지키면 교체 가능하다. 예: 플러그인 커넥터는 `{ id, kind, scope, govTier, status, summary, fetch() }` 형태만 맞추면 레지스트리에 push 하는 것으로 추가된다.

## 3. 히어로 케이스 전략

깊이는 **JBG-104 전주 중앙로 카페**(소상공인, 금리 부담 + 매출 둔화 → 정책금융 검토)에 집중한다. 이 한 케이스에서 고객 추적 → 근거 수집(플러그인) → 스킬 실행 → 산출물 생성 → 사람 승인 → 감사까지 **자율운영 전체 루프**가 끝까지 보이도록 만든다. 나머지 케이스는 동일 구조의 경량 버전.

## 4. 데이터 거버넌스 = 척추

외부 프런티어 LLM API로 에이전트를 돌리되, 고객 원본 PII는 외부로 나가지 않는다. 4중 방어(등급제·토큰화·모델 라우팅·외부 반출 제한+감사)를 `07-data-governance-pii.md`에 정의하고, 모든 플러그인/스킬/에이전트 호출이 이 계층을 통과하도록 한다. 데모는 거버넌스 패널에서 토큰화 전/후와 모델 라우팅을 가시화한다.

## 5. 오늘 구현 우선순위

1. 설계 문서 세트(본 폴더)
2. 데이터 거버넌스 계층 + 패널
3. 히어로 케이스 자율운영 상세 뷰 + 산출물 MD
4. 플러그인 레지스트리 페이지(리서치 기반 실제 내용)
5. 실질 스킬 콘텐츠 3-5개 + 보기/수정
6. 고객 DB/추적 + 보드 훅
7. UX/UI 수리 + 스크롤 수정
8. README 다이어그램·스크린샷

## 6. 비범위 (YAGNI)

- 실제 백엔드/DB 연결 (정적 데모 유지)
- 실제 외부 MCP 라이브 호출 (커넥터는 연결 가능 구조 + 모의 응답)
- 로그인/권한 시스템 전면화 (역할 표시 수준만)
