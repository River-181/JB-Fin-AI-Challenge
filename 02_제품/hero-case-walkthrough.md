## 자율 운영 루프 (단계별 · 에이전트 · 근거 · 산출 · 통제)

### 1단계 — 위험 신호 수집·등록 (L0)
- **위험신호 조기감지 에이전트**가 기사·정책·상담 노트에서 금리 부담·매출 둔화 신호를 수집 → Case 생성, Evidence 연결(`evidenceIds: jb-ai-mou, smallbiz-burden, rate-shock, digital-gap`).
- 근거 등급 칩: 공공데이터/시뮬레이션/추정 구분. 시장 근거: 자영업자 취약 42.7만·연체율 11.16%(취약).
- 통제: 관찰·로그만, 감사 원장 기록.

### 2단계 — 상환위험 분해 (L1)
- **상환위험 분류 에이전트**(`cashflow-stress`,`rate-relief`,`risk-banding`)가 금리 민감도·매출 둔화·연체 조짐을 분해, **판단·근거·confidence·다음 확인 항목**을 남김.
- 점수 분해(`computeRiskDecision`)로 "왜 88점인가"를 설명(설명가능성).

### 3단계 — 정책금융 매칭 (L1)
- **정책금융 매칭 에이전트**(`policy-match`,`document-checklist`,`eligibility-explain`)가 대환/정책금융 후보와 **필요 서류 체크리스트** 생성. 최소 1개 이상 제도/요건 단서 첨부(기능명세 결정 규칙).

### 4단계 — 고객 안내 초안 (L1→L2)
- **RM 보좌 에이전트**(`notification-brief`,`tone-control`,`next-best-action`)가 RM 콜백 초안·내부 메모 생성. 산출물은 케이스 상세의 **MD deliverable**로 저장.

### 5단계 — 데이터 거버넌스 (외부 LLM 호출 구간)
- 외부 모델로 비식별 요약을 보낼 때 **① 등급제 → ② PII 토큰화 → ③ 모델 라우팅(국내/외부) → ④ 반출 스캔**을 통과. 원본 PII는 국내 볼트에만.
- 근거: 신용정보법 §40조의2(특별법)·PIPA §28조의4/§28조의5. 패널에서 토큰화 전/후·라우팅·스캔 결과 가시화.

### 6단계 — 준법 검토 (L1)
- **준법 검토 에이전트**(`compliance-guard`,`privacy-redaction`,`claim-limiter`)가 금융조건 확정 표현 금지·과장·PII 노출을 검토. 게이트: `["금융조건 확정 표현 금지","passed"]` 등.
- 반려 시 4단계로 검토 루프(재작성).

### 7단계 — 승인 게이트 + 감사 (L2)
- riskScore 88 → 외부 행동은 승인 전 차단. **RM 최종 승인자**가 승인(`approveCase`) → 상태 `Approved`.
- **감사 원장**(`auditChainRecords`)에 actor·action·evidence id·approval id·시각을 무결성 해시로 기록(책임소재).

### 8단계 — 후속관리 (L0)
- 승인 후 콜백 태스크·사후관리 큐 등록, Activity Feed·Properties Panel 갱신. 사후관리 누락 0건 목표.

## 이 케이스가 증명하는 것 (심사 직접 답변)
- **3.1 판단·행동·검증 구조**: 8단계가 판단→행동 초안→검증(준법)→승인→감사→후속으로 닫힌 루프.
- **4.2 동작 MVP**: 위 흐름이 `?demo=sme`에서 클릭으로 재현(케이스 선택·AgentRun·승인·로그 갱신).
- **2.4 / 5.5 리스크**: PII 비반출 거버넌스·승인 게이트·감사 원장·환각 방지(근거/불확실성 의무 출력).
- **2.3 고객가치**: triage 시간 단축·정책금융 기회 발굴·사후관리 누락 방지.
