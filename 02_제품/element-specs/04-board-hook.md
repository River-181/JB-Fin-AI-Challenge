# 04 · 케이스 보드 처리 훅

> 모듈: 칸반 drop 핸들러 · "케이스를 옮겨서 일을 처리 → 실제 훅"

## 개념
케이스 보드(칸반)에서 케이스를 다른 컬럼으로 옮기면 단순 상태 변경이 아니라 **실제 업무 처리 훅**이 발동한다: 상태 전이 + 에이전트 작업 + 산출물 생성 + 감사 기록.

## 컬럼 ↔ 처리
| 컬럼(stage) | drop 시 훅 |
| --- | --- |
| `todo` → `in_progress` | 배정 에이전트 실행 시작, 활동 로그 |
| `in_progress` → `pending_approval` | 산출물 초안 생성(03), 승인 게이트 활성 |
| `pending_approval` → `done` | 사람 승인 처리, 감사 기록, 산출물 확정 |
| → `blocked` | do-not-contact/사기 차단, 외부 행동 금지 |

## 동작
- 기존 `moveCaseToColumn(id, status)` 확장: 전이 시 `runHook(case, fromStage, toStage)` 호출
- `runHook`은 거버넌스(07) 경유, 산출물(03)·감사·활동 갱신, 토스트 피드백
- 잘못된 전이(예: 승인 없이 done)은 가드로 차단 + 안내

## 갈아끼우기
훅 매핑은 `{from→to: handler}` 테이블. 새 처리 규칙 추가는 테이블 항목 추가.

## 심사 효과
④ 운영 루프가 손으로 만질 수 있는 형태로 작동(인터랙티브 검증).
