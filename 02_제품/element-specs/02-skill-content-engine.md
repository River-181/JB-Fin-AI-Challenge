# 02 · 실질 스킬 콘텐츠 엔진 + 보기/수정

> 모듈: `skillRegistry[].body` · 핵심 스킬에 실제 운영 콘텐츠(수십~수백 줄)

## 문제
스킬이 한 줄 라벨 + purpose뿐. 실제 은행 업무에서 쓸 법한 **실질 내용**(절차·입력·출력·근거 법령/정책·판단 기준·예시)이 들어가야 하고, 사용자가 보거나 수정할 수 있어야 한다.

## 범위 (전부 말고 핵심 몇 개)
히어로 케이스(전주 카페 SME)에 직접 쓰이는 스킬 우선 심화:
1. `cashflow-stress` — 매출 둔화·금리 부담·상환 압박 판단
2. `policy-match` — 정책금융·대환·보증 매칭 (리서치 기반 실제 프로그램)
3. `compliance-guard` — 과장표현·개인정보·준법 검토 (현행법 근거)
4. `document-checklist` — 상담 전 서류·확인 질문
5. `audit-ledger` — 근거·판단·행동·승인 기록

## 콘텐츠 구조 (body, markdown)
각 스킬 body에 아래 섹션:
- **목적/적용 상황**
- **입력** (필요 데이터 + 데이터 등급 → 거버넌스 연동)
- **처리 절차** (단계별, 수십 줄)
- **판단 기준/임계값** (예: 전세가율 80% 초과 경고, DSR, 금리인하요구권 요건)
- **출력** (산출물 형식 → 03 deliverables 연동)
- **근거 법령·정책** (플러그인 law/policy 커넥터 인용, 리서치 출처)
- **예시** (히어로 케이스 적용 예)
- **사람 승인 필요 여부 / 리스크 등급**

## 데이터 인터페이스
```js
{ slug, type, purpose, approval, risk, enabled,
  body: "## 목적 ...",      // markdown, 신규
  inputs: [{field, tier}],  // 거버넌스용
  sources: ["law-moleg#개인정보보호법", "policy-sema#정책자금"] }
```

## 보기/수정 UI
- 스킬 상세 뷰에 **본문(md 렌더) + [편집] 버튼**
- 편집 → textarea로 body 수정 → 저장(localStorage 반영). 데모에서 운영자가 스킬을 직접 다듬는 모습.

## 갈아끼우기
스킬 = 레지스트리 객체. body/sources 교체로 내용 갱신, 새 스킬 추가는 push.

## 심사 효과
② 금융 실무성, ③ 에이전트 역량의 구체성.
