# 03 · 케이스별 산출물 (MD 결과물) + 뷰어

> 모듈: `deliverableRegistry[caseId]` · "일을 시켰으면 실제 결과물"

## 개념
에이전트가 케이스를 처리하면 실제 업무 결과물이 나와야 한다. 각 케이스의 주요 산출물을 **MD 파일 형태**로 보유하고, 케이스 상세에서 뷰어로 본다. 실제 RM이 받는 결과물처럼 보이게 한다.

## 히어로 케이스(전주 카페) 산출물 예
1. **상환 스트레스 진단 리포트** — 매출 추이, 금리 부담, DSR, 상환 시나리오
2. **정책금융 매칭 결과** — 적합 프로그램(소진공 정책자금·대환·보증), 한도/금리/요건, 근거 출처
3. **RM 콜백 스크립트 초안** — 고객 안내 멘트(준법 검토 통과, 과장표현 제거)
4. **상담 전 서류 체크리스트** — 필요서류·확인질문
5. **준법·감사 요약** — 거버넌스 처리(토큰화/라우팅), 승인 게이트, 감사 해시

## 데이터 인터페이스
```js
deliverableRegistry = {
  "jeonju-cafe": [
    { id, title, kind:"report|script|checklist|audit",
      status:"generated|draft|pending",
      generatedBy:"cashflow-triage", at:"2026-06-14 10:20",
      body:"# 상환 스트레스 진단 ...",   // markdown
      govNote:"PII 토큰화 후 외부 모델 초안 → 사람 검토" }
  ]
}
```

## UI
- 케이스 상세 "산출물" 섹션: 카드 목록(제목·종류·생성 에이전트·상태)
- 카드 클릭 → MD 렌더 뷰어(모달 또는 패널) + 다운로드
- 미생성 상태에서 `산출물 생성` 버튼 → 상태 전이 + 산출물 표시(데모 시드)

## 보드 훅 연동 (04)
보드에서 케이스를 "처리" 컬럼으로 옮기면 해당 산출물 생성이 트리거된다.

## 갈아끼우기
산출물은 케이스ID 키 레지스트리. body는 md 문자열 → 실제 생성 엔진으로 교체 가능.

## 심사 효과
④ 완성도·검증가능성의 핵심: 심사자가 "실제 결과물"을 눈으로 확인.
