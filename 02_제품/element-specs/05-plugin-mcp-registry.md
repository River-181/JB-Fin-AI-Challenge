# 05 · 플러그인 / MCP 레지스트리 페이지

> 모듈: `pluginRegistry[]` + `pluginsPage()` · 에이전트가 외부 데이터에 접근하는 연결점

## 목적
에이전트가 실제로 일하려면 외부 데이터에 접근해야 한다. 법령·정책·뉴스·JB 금융 DB를 **모듈식 커넥터**로 등록하고, 연결 상태·스코프·거버넌스 등급을 보여준다. 실제 라이브 호출 대신 "연결 가능한 구조 + 모의 응답 + 실제 리서치 기반 내용"으로 시연한다.

## 커넥터 종류 (kind)
| kind | 예시 커넥터 | 용도 (히어로=전주 카페) |
| --- | --- | --- |
| `law` | 국가법령정보 MCP | 개인정보보호법·신용정보법·여신전문금융업법 조문 |
| `policy` | 소상공인 정책금융 / 국회 의안 / 금융위 정책 | 정책자금·대환·보증 프로그램 매칭 |
| `news` | 지역경제·상권 뉴스 피드 | 전주 원도심 상권/재개발/경기 신호 |
| `realestate` | 재개발·도시정비 정보 | 상권 변화·임대료 영향(전세 케이스 시 등기·시세) |
| `jb-db` | JB 금융 데이터베이스 MCP | 고객 거래·상담·심사 이력 (PII → 거버넌스 필수) |

## 데이터 인터페이스 (swappable)
```js
{
  id: "law-moleg",
  kind: "law",
  name: "국가법령정보 커넥터",
  scope: ["개인정보보호법","신용정보법","여신전문금융업법"],
  govTier: "public",          // PII 없음 → 외부 모델 허용
  status: "connected|available|blocked",
  summary: "현행법 조문·시행령 조회",
  sample: [...],              // 리서치 기반 실제 항목
  fetch(query) -> results     // 데모는 모의/캐시 응답
}
```
- `govTier`가 `jb-db`처럼 PII를 포함하면 거버넌스 계층(07)이 토큰화·온프레 라우팅을 강제한다.

## 페이지 UI (`pluginsPage`)
- 커넥터 카드 그리드: 종류 아이콘·이름·스코프 칩·상태·거버넌스 등급 배지
- 커넥터 클릭 → 상세: 제공 데이터 샘플(실제 리서치 내용), 어떤 스킬/에이전트가 사용하는지, 거버넌스 처리 방식
- `연결`/`연결 해제` 토글(데모 상태), `테스트 조회` 버튼(모의 응답)

## 갈아끼우기
새 커넥터 = 레지스트리에 객체 push. 페이지·케이스 상세의 근거 패널이 자동 반영.

## 심사 효과
② 금융·사업 연계(실데이터 접근 경로), ⑤ 확장성(커넥터 추가만으로 도메인 확장).
