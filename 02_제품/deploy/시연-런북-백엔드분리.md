# 시연 런북 — 백엔드 물리적 분리 (Docker)

목적: "원본 PII 존은 외부 인터넷에 물리적으로 나갈 수 없다"를 심사위원 앞에서 30초 안에 증명한다.
구성: [`docker-compose.yml`](./docker-compose.yml) — `console`(정적 UI + API 프록시, edge+pii-net) / `pii-zone`(Ollama, pii-net만·internal).

## B안: 분할화면 녹화 레시피

- **좌측**: 브라우저 — 콘솔 UI, `http://127.0.0.1:8000/index.html?live=1`
- **우측**: 터미널 2분할
  - 위: `tail -f 02_제품/scripts/llm-runs.jsonl | jq .` (LLM 호출 원장 실시간)
  - 아래: `watch -n1 docker compose -f 02_제품/deploy/docker-compose.yml ps` (컨테이너 상태)

## 물리 분리 증명 시퀀스 (복붙용)

```bash
cd 02_제품/deploy

# ① 기동
docker compose up -d

# ② pii-net이 internal(외부 라우팅 불가)임을 확인
docker network inspect deploy_pii-net --format '{{.Internal}}'   # → true

# ③ 킬러 컷: PII존에서 바깥으로 나가려는 시도는 실패해야 한다
docker compose exec pii-zone curl -m 3 https://example.com
# → curl: (28) Connection timed out / 또는 DNS 실패 — "인터넷이 물리적으로 없다"

# ④ 콘솔 → PII존 내부 통신은 정상(같은 pii-net 안에서만 가능)
docker compose exec console curl http://pii-zone:11434/api/tags
# → 200 OK (모델 목록 응답)

# ⑤ 콘솔에서 케이스 실행 → 원장에 레코드 찍히는 장면
#    (좌측 브라우저에서 케이스 액션 클릭 → 우측 tail -f 창에 JSONL 한 줄 추가)

# ⑤-보조: 게이트웨이 직접 호출로 로컬모델 실동작 확인
curl -s -X POST http://127.0.0.1:8020/llm -H 'content-type: application/json' \
  -d '{"prompt":"한 문장으로: 전세가율이란?","engine":"ollama","caseId":"CCL-0001"}'
```

> **엔진 주의(Docker 모드)**: `console` 컨테이너 안에는 claude/codex CLI가 없다. Docker 시연에서는 반드시 `engine:"ollama"`로 호출한다 — claude/codex 폴백 사다리는 프록시를 호스트 네이티브로 돌릴 때만 작동한다. 반대로 이 제약이 서사가 된다: "PII 존 작업은 로컬모델로만 처리되고, 외부 프런티어는 컨테이너 밖(edge)에서만 접근 가능."

## 캡처 포인트 (발표 자료용 스틸컷 3장)

1. **③의 실패 화면** — `curl: (28) Connection timed out`. 이 한 장이 "물리적 분리"의 증거.
2. **`docker network inspect` 결과 JSON** — `"Internal": true` 가 보이는 터미널 캡처.
3. **좌(콘솔 케이스 실행) + 우(jsonl 원장에 줄 추가되는 순간)** 동시 프레임 — 분할화면 그대로.

## 실패 대비 (폴백)

Docker가 심사장 노트북에서 안 뜨는 경우, 발표 직전에 재현 리허설 없이 즉흥 대응하지 말고 이 컷으로 대체한다:

- 호스트에서 네이티브로 실행: `npm run dev` (8000) + `npm run demo:proxy` (8020, 키 없으면 폴백값).
- 물리 분리 증거는 "같은 컷"을 말로 설명하는 대신 미리 녹화해 둔 ③번 실패 화면 스크린샷/짧은 영상으로 대체 재생한다.
- "네트워크 설정(방화벽 아웃바운드 차단)으로 동일한 통제를 구현하며, Docker는 로컬 재현을 위한 편의 계층일 뿐"이라는 한 문장으로 넘어간다.

## 참고: 모델 최초 주입

`pii-net`이 `internal: true`라 컨테이너 안에서 직접 `ollama pull`이 안 된다. `docker-compose.yml` 하단 주석의 절차(임시로 edge에 연결 → pull → 재분리)를 시연 리허설 전에 미리 한 번 실행해 두고, 실제 시연 때는 이미 받아둔 모델(`qwen2.5:0.5b` 또는 `gemma3:1b` — Mac Docker Desktop은 GPU 미지원이라 소형 모델 권장)을 그대로 쓴다.
