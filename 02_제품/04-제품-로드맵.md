## 1. MVP (현재) — 검증된 기반

canon §9 기준. Vanilla JS/CSS/HTML 정적 앱(`app/index.html`·`app/app.js`·`app/modules.js`·`app/styles.css`), `python3 -m http.server` 구동, 네트워크 API 없이 브라우저 상태로 운영 루프 재현.

- **재현되는 동작**: 자연어 지시 → AgentRun 생성, 케이스 보드 상태 이동, 승인/반려로 상태 변경·감사 로그 추가, 전세 진단 폼 계산, 에이전트/스킬/플러그인/근거 카드 상세 연결, Properties 패널 on/off·접기.
- **운영 계약 구현**: `Case → AgentRun → Agent → Skill → Evidence → Approval → Audit`이 실제 상태 변화로 작동.
- **승인 레벨 엔진**: `computeRiskDecision` → `approvalLevelFor`로 L0~L4 라우팅.
- **검증**: `scripts/verify_static.py`, `node --check app/*.js`, Playwright e2e 19종, 골든 패스 3종.
- **거버넌스 가시화**: 플러그인 커넥터의 `restricted`(PII) 등급은 토큰화+국내·온프레 라우팅+반출 스캔 강제를 UI로 표현.

## 2. 본선 고도화 — 함수 계약의 백엔드 승격

정적 함수 계약을 서버 API로 1:1 전환하고 실제 추론·검색·규칙 엔진을 결합한다(canon §9, 기능명세 §5).

- **백엔드 승격**: `computeRiskDecision`·`buildDashboardData`·`auditChainRecords`·`moveCaseToColumn` 등 정적 함수를 서버 API로 1:1 승격. UI 계약 불변, 데이터 소스만 교체.
- **RAG 도입**: 공공·내부 문서(법령·정책·상담 노트) 검색 기반 근거 생성으로 Evidence를 실데이터화. 현재 플러그인 커넥터(law·policy·news·jb-db)를 실제 검색 백엔드에 연결.
- **Rule Engine**: 승인 레벨 매트릭스(L0~L4)와 조치 유형 라우팅을 규칙 엔진으로 외부화해 정책 변경을 코드 수정 없이 반영.
- **멀티 에이전트 오케스트레이션**: 운영 조율 에이전트가 14종 전문 에이전트에 케이스 맥락·스킬·승인 게이트를 분배하는 실제 오케스트레이션.
- **모델 라우팅**: 국내·온프레 모델(원본 PII)과 외부 프런티어 모델(토큰 입력만) 라우팅 운영. 거버넌스 게이트를 모든 외부 호출의 실 관문으로.
- **감사 무결성**: 감사 원장에 무결성 해시 적용, 감사 JSON 내보내기를 서명 기반으로.

## 3. PoC — 실데이터 연동·제한 케이스 검증

- **데이터 연동**: 한국은행 ECOS(금리·경제지표), HUG·국토교통부 실거래가·전세보증, 정책금융 카탈로그(소진공/중기부) 연결. 등기/실거래는 자동 대량수집 제약을 고려해 사람 트리거+캐시 방식(기능명세 §5·부록 A).
- **라이선스 준수**: 외부 데이터·API의 출처·라이선스·제약을 인벤토리로 관리(공공누리 유형, 호출 제한, 상용 API 약관). 외부 LLM에 원본 PII 전송 금지를 코드 게이트로 강제.
- **검증 범위**: 골든 패스 3종(소상공인 자금압박·전세 보호·보이스피싱)을 실데이터로 좁게 검증해 판단 정확도·근거 추적성·차단 정확성을 측정.

## 4. 파일럿 — 1개 본부 운영 검증

- **범위**: 1개 본부/지점 단위 RM·준법 담당자 대상 파일럿(기능명세 §5).
- **운영 루프 안정화**: 실 운영자 피드백으로 승인 레벨 임계값·스킬 본문·체크리스트 조정. SLA·사후관리 큐 운영.
- **통제 검증**: 사람 승인 전 자동 발송 0건, 고위험 fraud 외부 접촉 차단, 감사 추적성 확보(canon §3 KPI 방향).

## 5. 내부 적용 — 사후관리·심사 보조

- **적용 영역**: 기업여신 상담→심사→**사후관리** 전반의 AI 보조. 문서 데이터 추출·재무자료 요약 심사 지원(2025-12-26 JB금융그룹–네이버클라우드 MOU·공표 추진 방향과 연결. "이미 운영 중"이 아닌 추진 방향으로만 표현).
- **업무 통합**: 케이스·승인·감사를 기존 RM 업무 시스템과 연동, 사후관리 누락 0건 목표.

## 6. 상용 — 계열사·업무·고객군 확장

- **계열사 확장**: 전북은행 외 광주은행·JB우리캐피탈로 조직 프로필 확장(설정 화면의 조직 전환을 실 멀티테넌시로).
- **업무 영역 확장**: 기업여신 심사 보조, WM(자산관리)로 스킬 추가.
- **고객군 확장**: 소상공인·전세 고객에서 중소기업·가계로. 스킬 저장소에 신규 스킬 패키지 추가 방식으로 점진 확장.
- **전략 정렬**: 그룹 "AX(AI 전환) 원년" 전략 및 지역 금융(전북·광주/전남) 접점 강화와 부합.

## 리스크 대응 방향 (심사 5.5)

- **개인정보·보안**: 데이터 등급제·토큰화·모델 라우팅·반출 스캔 4중 방어. 신용정보법 우선 적용 인식(canon §4).
- **환각·설명가능성**: 에이전트는 결론·근거·불확실성·다음 확인 항목을 함께 남김. 확정/보장 표현은 준법 검토 에이전트가 차단.
- **책임 소재**: 고객 대상 행동은 사람 승인 전 차단, 모든 승인/반려/차단 actor·시각을 감사 원장에 기록.
